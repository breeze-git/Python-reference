# Определение и вызов

**Функция** — это именованный блок кода, **который выполняет определённую задачу** и может быть вызван (исполнен) **из любой части программы**.

**Синтаксис:**

```py
def function_name(): # заголовок функции
    блок кода # тело функции
```

Функции объявляются **ключевым словом `def`** (от англ. define – определять). За ключевым словом `def` следуют **название функции, круглые скобки `()` и двоеточие `:`.**

Первая строка объявления функции называется **заголовком функции**, а [блок кода](glossary.md#блок-кода), следующий после заголовка — **телом функции.**

Чтобы исполнить код функции, нужно ее вызвать. **Синтаксис вызова функции:**

```py
function_name()
```

## Именование функций

**Название функции:**

- должно начинаться с буквы или со знака подчеркивания.
- может содержать буквы, цифры и знак подчеркивания.
- не должно совпадать с названиями ключевых слов языка Python.
- применяется два типа наименования функций: camel case и underscore notation.
- чувствительно к регистру.

Рекомендуется использовать в именах функций глаголы, описывающие действие выполняемое функцией. Например: `calc_nums()`, `check_pass()`, `create_list()` и т.д.

## Вызов функций

Для того, чтобы вызвать функцию нам необходимо прописать ее название и круглые скобки. **Объявление функции должно предшествовать ее вызову.**

Пример объвления и вызова функции:

```py
def print_hello():
    print('Hello World!')

print_hello()

Hello World!
```

Иногда при объявлении функции требуется сделать своего рода заглушку, чтобы функция ничего не выполняла. Тогда мы используем ключевое слово `pass`:

```py
def do_nothing():
    pass
```

# Параметры и аргументы

## Основы параметров и аргументов

**Параметр** – это переменная, указываемая в круглых скобках в объявлении функции и принимающая **аргумент** для работы с ним в теле функции. В Python количество параметров у функций неограниченно.

```py
def my_func(param): # параметр
    print(param)
```

У функций есть **обязательные** и **необязательные** параметры.

Обязательные параметры — параметры, значения которых **должны быть переданы при вызове функции**. В случае пропуска этих параметров будет вызвана ошибка `TypeError`.

```py
def my_sqr(num, pwr):
    print(num**pwr)

my_sqr(3)

TypeError: my_sqr() missing 1 required positional argument: 'pwr'
```

**Необязательные параметры** — параметры, **имеющие значения по умолчанию**. Их можно не указывать — тогда будет использовано значение по умолчанию.

Чтобы задать значение **параметра по умолчанию**, в списке параметров функции достаточно после имени переменной **написать знак равенства и нужное значение**:

```py
def my_sqr(num, pwr=2):
    print(num**pwr)

my_sqr(3)

9
```

В функции все **необязательные** параметры должны идти **после** **обязательных**.

Значение по умолчанию для параметра создается **единожды при определении функции** (обычно при загрузке модуля) и становится атрибутом (свойством) функции. Поэтому, если значение по умолчанию **изменяемый объект**, то его изменение повлияет на каждый следующий вызов функции.

Это можно рассмотреть на примере функции `append()`, где в качестве значения по умолчанию используется **изменяемый тип данных** (список, тип `list`):

```py
def append(element, seq=[]):
    seq.append(element)
    return seq

print(append(10))
print(append(5))
print(append(1))

[10]
[10, 5]
[10, 5, 1]
```

---

**Аргумент** – это значение, которое передаётся функции при её вызове.

```py
my_func('Hello World!') # аргумент
```

В Python существует два вида аргументов: позиционные и именованные.

**Позиционные аргументы** - аргументы, передаваемые **без указания имён**. Их значения сопоставляется с параметрами функции по **позиции**.

```py
def my_sqr(num, pwr):
    print(num**pwr)

my_sqr(4, 2)

16
```

**Именованные аргументы** — аргументы, передаваемые с указанием **имени параметра**. Порядок упоминания именованных аргументов не имеет значения.

```py
def my_sqr(num, pwr):
    print(num**pwr)

my_sqr(4, pwr=2)

16
```

**Позиционные аргументы должны быть указаны до любых именованных.**

**Keyword-only аргументы** — аргументы, которые можно передавать **только по имени**. Чтобы задать их, в определении функции используют **символ `*`** как разделитель: он отделяет обычные аргументы (их можно указывать по имени и позиционно) от строго именованных.

```py
def my_sqr(num, *, pwr):
    print(num**pwr)

my_sqr(4, pwr=2) # pwr - keyword-only
```

Разделитель `*` можно использовать **только один раз в определении функции**. Его **нельзя** применять в функциях с неограниченным количеством позиционных аргументов `*args`.

## Переменное количество аргументов: *args, **kwargs

**Позиционные аргументы** можно получать в виде **кортежа `args`** с помощью оператора [распаковки](operators.md#операции-распаковки-и-упаковки-итерируемых-объектов-оператор--в-распаковке-и-упаковке) `*`.

```py
def mult(a, b, *args):
    res = a * b
    for n in args:
        res *= n
    return res

print(mult(1, 2, 3, 4))

24
```

Аргументы собираются **в кортеж `args` от текущей позиции до конца**. Это позволяет передавать функции **переменное количество** позиционных аргументов.

Параметр `args` в определении функции пишется **после позиционных параметров** перед первым параметром со значением по умолчанию.

---

**Именованные аргументы** можно получать в виде **словаря `kwargs`** с помощью двойного оператора распаковки `**`.

```py
def give_info(name, surname, age, **kwargs):
    print(f'name: {name}', 
          f'surname: {surname}', 
          f'age: {age}', 
          f'other: {kwargs}', 
          sep = '\n')

give_info('Will', 'Turner', 23, job='Pirate', gun='False')

name: Will
surname: Turner
age: 23
other: {'job': 'Pirate', 'gun': 'False'}
```

Аргументы **собираются в словарь `kwargs`**, где **ключ — имя**, а **значение — переданный аргумент**. Это позволяет передавать функции **переменное количество** именованных аргументов.

Параметр  `**kwargs` пишется **в самом конце**, после последнего аргумента со значением по умолчанию. При этом функция может содержать и `*args` и `**kwargs` параметры.

**Правильный порядок параметров:**

1. позиционные аргументы,
2. `*args` аргументы,
3. `**kwargs` аргументы.

# Возврат значений

Функции в Python могут возвращать результат с помощью оператора `return`.

**`return`** — это **оператор**, используемый внутри функции **для немедленного её завершения и передачи результата**(значения) в то место программы, **откуда функция была вызвана**.

```py
def sum_func(a, b):
    c = a + b
    return c

print(sum_func(3, 4))

7
```

Если после `return` указано выражение, оно вычисляется и возвращается как результат функции:

```py
def sum_func(a, b):
    return a + b

print(sum_func(10, 7))

17
```

Если `return` используется без значения или отсутствует, функция возвращает специальное значение [`None`](types.md#тип-nonetype):

```py
def print_hello():
    print('Hello World!')
    return

print(print_hello())

Hello World!
None
```

В функции может быть несколько вызовов `return`:

```py
def is_positive(num):
    if num > 0:
        return True
    else:
        return False

print(is_positive(1))

True
```

Обратите внимание, что в примере выше, второй вызов `return False` не выполнился, так как после первого `return` функция завершила работу и вернула результат.

**Как только выполнение доходит до `return` - функция останавливается и возвращает результат.**

В Python функции не ограничены возвратом всего одного значения. После инструкции `return` можно определить много выражений, разделенных запятыми:

```py
def sum_mult(a, b):
    return a + b, a * b

sumn, multn = sum_mult(10, 15)

print(sumn, multn)

25 150
```

При указании нескольких выражений через запятую Python автоматически упаковывает их в [кортеж](types.md#кортежи), который затем можно распаковать в переменные.

# Локальные и глобальные переменные

## Локальные переменные

**Локальные переменные** — переменные объявленные внутри функции и доступные только внутри этой функции.

```py
def my_func():
    s = 'Hello!'
    print(s)
```

В примере выше, `s` — это локальная переменная, недоступная за пределами функции `my_func()`.

То есть если написать такой код:

```py
def my_func():
    s = 'Hello!'

print(s)
```

Мы получим ошибку : `NameError: name 's' is not defined`

**Примечания:**

- Параметрические переменные также являются локальными переменными функции.

- Память для локальных переменных выделяется на время исполнения данной функции в специальной области, называемой стеком. При завершении работы функции память освобождается, внутренние результаты работы функции не сохраняются от одного обращения к другому.

---

## Глобальные переменные

**Глобальные переменные** — переменные, объявленные на верхнем уровне скрипта, доступны во всех функциях этого файла — только для чтения.

```py
s = 'Hello!'

def my_func():
    print(s)

my_func()

Hello!
```

В примере выше, `s` — это глобальная переменная, поэтому мы можем иметь к ней доступ внутри функции `my_func()`.

Чтобы изменить глобальную переменную из функции, нужно использовать ключевое слово `global`:

```py
n = 10

def my_func():
    global n
    n += 3

my_func()

print(n)

13
```

Используя одно зарезервированное слово `global` , можно объявить сразу несколько переменных: `global x, y, z`.

Если создать внутри функции локальную переменную с именем глобальной, функция будет обращаться к этой локальной переменной по [правилу LEGB](#область-видимости).

---

**Не рекомендуется использовать глобальные переменные, по следующим причинам:**

- Глобальные переменные затрудняют отладку программы. Значение глобальной переменной может быть изменено любой инструкцией в программном файле. Если обнаружится, что в глобальной переменной хранится неверное значение, то придется отыскать все инструкции, которые к ней обращаются, чтобы определить, откуда поступает плохое значение. В программе с тысячами строк кода сделать это непросто.

- Функции, использующие глобальные переменные, обычно зависят от этих переменных. Если возникнет необходимость применить такую функцию в другой программе, скорее всего придется эту функцию перепроектировать, чтобы она не опиралась на глобальную переменную.

- Глобальные переменные затрудняют понимание программы. Глобальная переменная может быть модифицирована любой инструкцией в программе. При необходимости разобраться в какой-то части программы, использующей глобальную переменную, придется узнать обо всех других частях программы, обращающихся к этой глобальной переменной.

## Глобальные константы

Следует избегать использования глобальных переменных, но допускается применение глобальных констант.

**Глобальная константа** – глобальное имя, ссылающееся на **неизменное значение**.

Язык Python не позволяет создавать настоящие глобальные константы, но их можно имитировать при помощи глобальных переменных.

Если глобальная переменная не объявляется с использованием ключевого слова `global` внутри функции, то внутри этой функции вы никак не сможете изменить значение этой глобальной переменной.

# Область видимости

**Область видимости переменной** - это часть программы, в которой переменная доступна для чтения и/или изменения.

В Python есть четыре уровня областей видимости.

**L — Local (локальная):**

- Область внутри текущей функции.
- Переменные, созданные внутри функции, существуют только в ней.
- Уничтожаются после завершения функции.

**E — Enclosing (обрамляющая / нелокальная):**

- Относится к вложенным функциям.
- Если переменная не найдена в локальной области, Python ищет её в области внешней (обрамляющей) функции.
- Чтобы изменить такую переменную из внутренней функции, нужно использовать ключевое слово `nonlocal`.

**G — Global (глобальная):**

- Область всего модуля (файла).
- Переменные, созданные на верхнем уровне скрипта, доступны во всех функциях этого файла — только для чтения.
- Чтобы изменить глобальную переменную из функции, нужно использовать ключевое слово `global`.

**B — Built-in (встроенная):**

- Содержит имена встроенных функций и объектов: len, print, range, int и т.д.
- Это "фоновый" уровень — Python обращается к нему, если имя не найдено в L, E, G.

**Таким образом формируется правило поиска LEGB.**

**Правило поиска LEGB:**

Когда Python встречает имя (например, x), он ищет его в таком порядке:

1. `Local` — внутри текущей функции,
2. `Enclosing` — во внешних функциях (если есть вложенность),
3. `Global` — на уровне модуля,
4. `Built-in` — среди встроенных имён.
5. Если имя не найдено нигде — возникает `NameError`.

# Функции высшего порядка

Любая **функция в языке Python** — **объект** типа `function`.

Работать с функциями можно как и с другими объектами: записывать их в переменные, передавать в качестве аргументов другим функциям, возвращать из функций и т.д.

**Функции высшего порядка** – это функции, которые принимают **другие функции в качестве аргументов и/или возвращают функции** в качестве результата.

Например напишем свою функцию высшего порядка `map()`:

```py
def sqr(item):
    return item**2

def map(func, items):
    return [func(item) for item in items]

numbers = [2, 4, 1, 7, 10]

print(map(sqr, numbers))

[4, 16, 1, 49, 100]
```

Функции результатом своей работы могут возвращать другие функции. Это позволяет нам писать генераторы функций, возвращающие функции в зависимости от передаваемых им аргументов.

```py
def generator(name):
    def hello():
        print(f'Hello from function, {name}!')
    return hello

func = generator('Will')

func()

Hello from function, Will!
```

Обратите внимание на то, что внутренняя функция `hello()` использует параметр внешней функции `generator()`. Такую вложенную функцию называют **замыканием**.

**Замыкания** – вложенные функции, ссылающиеся на переменные, объявленные **вне определения этой функции, и не являющиеся её параметрами**.

# Анонимные функции

**Анонимные функции (лямбда-функции)** – это короткие однострочные функции с использованием оператора `lambda`.

**Синтаксис анонимной функции:**

`lambda список_параметров: выражение`

где `список_параметров` – список параметров через запятую, `выражение` – значение, либо код, дающий значение.

В теле анонимной функции можно использовать [тернарный условный оператор](control_flow.md#тернарный-условный-оператор).

Анонимные функции имеет смысл использовать в таких случаях как:

- однократное использование функции;
- передача функций в качестве аргументов другим функциям;
- возвращение функции в качестве результата другой функции.

Разберем каждый случай на примере.

**Однократное использование функции:**

```py
points = [(1, -1), (2, 3), (-10, 15), (10, 9), (7, 18), (1, 5), (2, -4)]

print(sorted(points, key=lambda point: point[1])) # сортируем по второму значению кортежа
```

**Передача анонимных функций в качестве аргументов другим функциям:**

```py
nums = [7, 1, 13, 3, 10, 2, 4, 8]

print(list(map(lambda x: x**3, nums)))

[343, 1, 2197, 27, 1000, 8, 64, 512]
```

**Возвращение функции в качестве результата другой функции:**

```py
def sqr(num):
    return lambda x: x**2
```

**Анонимные функции поддерживают все способы передачи аргументов:**

- позиционные аргументы;
- именованные аргументы;
- переменный список позиционных аргументов (`*args`);
- переменный список именованных аргументов (`**kwargs`);
- keyword-only аргументы (*).

**Особенности и ограничения анонимных функций в Python:**

- в теле анонимной функции такие операторы, как return, pass, assert или raise, вызовут исключение SyntaxError;
- анонимная функция пишется как одна строка исполнения;
- анонимная функция может быть немедленно вызвана.

# Docstrings — строки документации

**Docstrings** — это встроенный в Python способ документировать код. В отличие от обычных комментариев через `#`, докстринги сохраняются в атрибуте объекта и доступны во время выполнения программы.

```py
def triangle_square(b, h):
    '''
    Calculates triangle square. 

    Args:
        b : base of triangle;
        h : height of triangle.

    Returns:
       number: triangle square. 
    ''' 
    return (b * h)/2
```

**Основные правила оформления docstrings:**

- Docstrings пишутся сразу после заголовка функции, класса или метода (на первой строке тела).
- Оборачиваются в тройные кавычки `"""текст"""` (даже если текст в одну строку).
- Записываются в форме многострочной строки, где первая строка начинается с заглавной буквы и
заканчивается точкой. Вторая строка оставляется пустой, а подробное описание начинается с третьей.

Доступ к строке документации функции можно получить с помощью атрибута этой функции `__doc__`. По такому принципу работает функция [`help()`](builtins.md#функции-отладки) в Python.
