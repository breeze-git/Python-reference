# Определение и вызов

**Функция** — это именованный блок кода, **который выполняет определённую задачу** и может быть вызван (исполнен) **из любой части программы**.

**Синтаксис:**

```py
def function_name(): # заголовок функции
    блок кода # тело функции
```

Функции объявляются **ключевым словом `def`** (от англ. define – определять). За ключевым словом `def` следуют **название функции, круглые скобки `()` и двоеточие `:`.**

Первая строка объявления функции называется **заголовком функции**, а [блок кода](glossary.md#блок-кода), следующий после заголовка — **телом функции.**

Чтобы исполнить код функции, нужно ее вызвать. **Синтаксис вызова функции:**

```py
function_name()
```

## Именование функций

**Название функции:**

- должно начинаться с буквы или со знака подчеркивания.
- может содержать буквы, цифры и знак подчеркивания.
- не должно совпадать с названиями ключевых слов языка Python.
- применяется два типа наименования функций: camel case и underscore notation.
- чувствительно к регистру.

Рекомендуется использовать в именах функций глаголы, описывающие действие выполняемое функцией. Например: `calc_nums()`, `check_pass()`, `create_list()` и т.д.

## Вызов функций

Для того, чтобы вызвать функцию нам необходимо прописать ее название и круглые скобки. **Объявление функции должно предшествовать ее вызову.**

Пример объвления и вызова функции:

```py
def print_hello():
    print('Hello World!')

print_hello()

Hello World!
```

Иногда при объявлении функции требуется сделать своего рода заглушку, чтобы функция ничего не выполняла. Тогда мы используем ключевое слово `pass`:

```py
def do_nothing():
    pass
```

# Параметры и аргументы

**Параметр** – это переменная, указываемая в круглых скобках в объявлении функции и принимающая **аргумент** для работы с ним в теле функции.

```py
def my_func(param): # параметр
    print(param)
```

**Аргумент** – это значение, которое передаётся функции при её вызове.

```py
my_func('Hello World!') # аргумент
```

**В Python количество параметров у функций неограниченно.**

**Пример объявления и вызова функции с параметрами:**

```py
def print_hello(name):
    print('Hello,', name)

print('Will')

Hello, Will
```

# Возврат значений

Функции в Python могут возвращать результат с помощью оператора `return`.

**`return`** — это **оператор**, используемый внутри функции **для немедленного её завершения и передачи результата**(значения) в то место программы, **откуда функция была вызвана**.

```py
def sum_func(a, b):
    c = a + b
    return c

print(sum_func(3, 4))

7
```

Если после `return` указано выражение, оно вычисляется и возвращается как результат функции:

```py
def sum_func(a, b):
    return a + b

print(sum_func(10, 7))

17
```

Если `return` используется без значения или отсутствует, функция возвращает специальное значение [`None`](types.md#тип-nonetype):

```py
def print_hello():
    print('Hello World!')
    return

print(print_hello())

Hello World!
None
```

В функции может быть несколько вызовов `return`:

```py
def is_positive(num):
    if num > 0:
        return True
    else:
        return False

print(is_positive(1))

True
```

Обратите внимание, что в примере выше, второй вызов `return False` не выполнился, так как после первого `return` функция завершила работу и вернула результат.

**Как только выполнение доходит до `return` - функция останавливается и возвращает результат.**

В Python функции не ограничены возвратом всего одного значения. После инструкции `return` можно определить много выражений, разделенных запятыми:

```py
def sum_mult(a, b):
    return a + b, a * b

sumn, multn = sum_mult(10, 15)

print(sumn, multn)

25 150
```

При указании нескольких выражений через запятую Python автоматически упаковывает их в [кортеж](types.md#кортежи), который затем можно распаковать в переменные.

# Локальные и глобальные переменные

## Локальные переменные

**Локальные переменные** — переменные объявленные внутри функции и доступные только внутри этой функции.

```py
def my_func():
    s = 'Hello!'
    print(s)
```

В примере выше, `s` — это локальная переменная, недоступная за пределами функции `my_func()`.

То есть если написать такой код:

```py
def my_func():
    s = 'Hello!'

print(s)
```

Мы получим ошибку : `NameError: name 's' is not defined`

**Примечания:**

- Параметрические переменные также являются локальными переменными функции.

- Память для локальных переменных выделяется на время исполнения данной функции в специальной области, называемой стеком. При завершении работы функции память освобождается, внутренние результаты работы функции не сохраняются от одного обращения к другому.

---

## Глобальные переменные

**Глобальные переменные** — переменные, объявленные на верхнем уровне скрипта, доступны во всех функциях этого файла — только для чтения.

```py
s = 'Hello!'

def my_func():
    print(s)

my_func()

Hello!
```

В примере выше, `s` — это глобальная переменная, поэтому мы можем иметь к ней доступ внутри функции `my_func()`.

Чтобы изменить глобальную переменную из функции, нужно использовать ключевое слово `global`:

```py
n = 10

def my_func():
    global n
    n += 3

my_func()

print(n)

13
```

Если создать внутри функции локальную переменную с именем глобальной, функция будет обращаться к этой локальной переменной по [правилу LEGB](#область-видимости).

---

**Не рекомендуется использовать глобальные переменные, по следующим причинам:**

- Глобальные переменные затрудняют отладку программы. Значение глобальной переменной может быть изменено любой инструкцией в программном файле. Если обнаружится, что в глобальной переменной хранится неверное значение, то придется отыскать все инструкции, которые к ней обращаются, чтобы определить, откуда поступает плохое значение. В программе с тысячами строк кода сделать это непросто.

- Функции, использующие глобальные переменные, обычно зависят от этих переменных. Если возникнет необходимость применить такую функцию в другой программе, скорее всего придется эту функцию перепроектировать, чтобы она не опиралась на глобальную переменную.

- Глобальные переменные затрудняют понимание программы. Глобальная переменная может быть модифицирована любой инструкцией в программе. При необходимости разобраться в какой-то части программы, использующей глобальную переменную, придется узнать обо всех других частях программы, обращающихся к этой глобальной переменной.

## Глобальные константы

Следует избегать использования глобальных переменных, но допускается применение глобальных констант.

**Глобальная константа** – глобальное имя, ссылающееся на **неизменное значение**.

Язык Python не позволяет создавать настоящие глобальные константы, но их можно имитировать при помощи глобальных переменных.

Если глобальная переменная не объявляется с использованием ключевого слова `global` внутри функции, то внутри этой функции вы никак не сможете изменить значение этой глобальной переменной.

# Область видимости

**Область видимости переменной** - это часть программы, в которой переменная доступна для чтения и/или изменения.

В Python есть четыре уровня областей видимости.

**L — Local (локальная):**

- Область внутри текущей функции.
- Переменные, созданные внутри функции, существуют только в ней.
- Уничтожаются после завершения функции.

**E — Enclosing (обрамляющая / нелокальная):**

- Относится к вложенным функциям.
- Если переменная не найдена в локальной области, Python ищет её в области внешней (обрамляющей) функции.
- Чтобы изменить такую переменную из внутренней функции, нужно использовать ключевое слово `nonlocal`.

**G — Global (глобальная):**

- Область всего модуля (файла).
- Переменные, созданные на верхнем уровне скрипта, доступны во всех функциях этого файла — только для чтения.
- Чтобы изменить глобальную переменную из функции, нужно использовать ключевое слово `global`.

**B — Built-in (встроенная):**

- Содержит имена встроенных функций и объектов: len, print, range, int и т.д.
- Это "фоновый" уровень — Python обращается к нему, если имя не найдено в L, E, G.

**Таким образом формируется правило поиска LEGB.**

**Правило поиска LEGB:**

Когда Python встречает имя (например, x), он ищет его в таком порядке:

1. `Local` — внутри текущей функции,
2. `Enclosing` — во внешних функциях (если есть вложенность),
3. `Global` — на уровне модуля,
4. `Built-in` — среди встроенных имён.
5. Если имя не найдено нигде — возникает `NameError`.
