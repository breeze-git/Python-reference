# Основные понятния

**<b id="#oop-definition">Объектно-ориентированное программирование (ООП)</b>** — парадигма программирования, в которой основными концепциями являются понятия **объектов** и **классов**.

**Класс** — абстрактный тип данных, выступающий в роли шаблона для создания объектов. Он определяет структуру и логику.

**Объект** — экземпляр, конкретное воплощение класса в памяти. Обладает состоянием и поведением.

В Python всё является объектами — числа, строки, коллекции, функции и т.д. Каждый тип данных это класс, на базе которого создаются объекты. В этом можно убедиться вызвав функцию `type()` на любом объекте.

```py
print(type("Hello, World!"))

<class 'str'>
```

# Синтаксис класса

Объекты могут хранить данные и реализовывать поведение с помощью **полей** и **методов**.

**Поля** — это переменные, принадлежащие объекту или классу. Получать доступ к **полям класса** и изменять их значения могут все **экземпляры этого класса**. В то время как доступ к **полям конкретного объекта** можно получить только через **этот объект**.

**Методы** — это функции, наследуемые объектом от своего класса. Обычно методы работают с полями своих объектов, но с помощью декоратора `@classmethod` они могут работать с полями самого класса.

Вместе поля и методы принято называть **атрибутами**.

```py
class User:  # объявление класса
    users_number = 0  # поле класса

    def __init__(self, name, email):  # функция конструктор класса
        self.name = name  # поля объекта
        self.email = email

        self.__class__.users_number += (
            1  # обращение к полю класса, через self.__class__
        )

    def info(self):
        print(f"name: {self.name}, email: {self.email}")  # метод объекта

    @classmethod
    def users_info(cls):
        print(f"Number of users: {cls.users_number}")


user = User("Jack", "sparrow@gmail.com")  # создание объектов
user1 = User("Will", "turner@gmail.com")

# вызов методов
user.info()  # name: Jack, email: sparrow@gmail.com
user1.info()  # name: Will, email: turner@gmail.com

# обращение к полю объекта
print(user.name)  # Jack

# обращение к полю класса
print(user.users_number)  # 2

# обращение к методу класса
user.users_info()  # Number of users: 2
```

**Метод `__init__`** — это конструктор класса. Он вызывается автоматически при создании нового объекта. В нём задаётся начальное состояние объекта — данные, с которыми он будет работать, то есть происходит инициализация объекта.

**Параметр `self`** — это ссылка на конкретный объект, которая автоматически передается методу в момент его вызова. Через нee данные сохраняются в этот объект.

**Параметр `cls`** — это ссылка на класс объекта, которая передается методу в момент его вызова через декоратор `@classmethod`. Таким образом метод класса может работать напрямую с полями своего класса.

Каждый объект может ссылаться на свой класс через атрибут `self.__class__`.

В примере выше два объекта создаются от одного класса. Они имеют одинаковые поля и поведение, но хранят разные данные.

**Примечание:**

- Стоит помнить, что создание одноименных полей у объекта и класса объекта, создает конфликт имен переменных, при котором новое поле объекта, перекрывает поле класса этого объекта. Вследствии чего получить доступ к полю класса, через этот объект будет невозможно.

# Основополагающие принципы ООП

## Наследование

**Наследование** — это механизм, позволяющий создать новый дочерний класс на основе уже существующего родительского класса. При этом дочерний класс автоматически наследует все поля и методы родителя, что позволяет избежать дублирования кода и выстраивать иерархии от общего к частному.

**Родительский класс** — класс, чьи свойства и методы наследуются.

**Дочерний класс** — класс, который наследует функционал и может его дополнять или изменять.

В Python при объявлении дочернего класса в скобках необходимо указать имя родительского.

```py
class User: # Родительский класс
    def __init__(self, name):
        self.name = name

    def info(self):
        print(f"User: {self.name}")

class Admin(User): # Дочерний класс, наследует User
    def __init__(self, name, permissions):
        # Вызов конструктора родителя через super()
        super().__init__(name) 
        self.permissions = permissions # Дополнительное поле

    def info(self): # Переопределение метода
        super().info() # Вызов логики родителя
        print(f"Status: Admin | Permissions: {self.permissions}")

admin = Admin("Root", "All access")

admin.info()

User: Root
Status: Admin | Permissions: All access
```

**Функция `super()`** — возвращает прокси-объект, который делегирует вызовы методов родительскому классу. Это необходимо, чтобы не переписывать логику родителя заново (например, в `__init__`), а дополнять её.

Если объект дочернего класса вызывает метод, Python сначала ищет его в дочернем классе и, если не находит — поднимается к родителю.

**Виды наследования:**

- **Многоуровневое наследование** — дочерний класс наследуется от другого дочернего класса, образуя цепочку наследования.

- **Множественное наследование** — Python позволяет одному классу наследоваться сразу от нескольких. Порядок поиска в этом случае определяется алгоритмом MRO (Method Resolution Order).

- **Гибридное наследование** — смешение различных видов наследования. Как и в случае с множественным наследованием порядок поиска определяется алгоритмом MRO.

**Посмотреть порядок поиска методов в цепочке наследования можно, введя:**

```py
print(some_class.mro())
```

## Инкапсуляция

**Инкапсуляция** — это принцип ООП, который заключается в объединении данных и методов, работающих с этими данными, в единый компонент — класс, и ограничении прямого доступа к ним извне.

**Цель инкапсуляции** — защитить внутреннее состояние объекта от некорректного изменения и скрыть детали реализации, предоставляя пользователю только необходимый интерфейс.

В Python нет строгих запретов на доступ. Ограничение доступа строится на соглашениях об именовании:

- **Публичные (Public)** — атрибуты без подчеркиваний (`self.name`), доступные отовсюду.

- **Защищенные (Protected)** — атрибуты с одним нижним подчеркиванием (`self._name`). Сигнал для разработчиков, что это внутренняя деталь и не стоит использовать ее вне класса. Технически доступ сохраняется, но прямое обращение считается плохим тоном.

- **Приватные (Private)** — атрибуты с двумя подчеркиваниями (`self.__name`). Python применяет механизм искажение имени — **Name Mangling**, чтобы максимально затруднить доступ к атрибуту извне.

**Механизм Name Mangling** — защищает атрибут от случайного перезаписывания в дочерних классах. Например когда объявляется атрибут `__password` в классе `User`, Python переименовывает его во внутреннюю переменную `_User__password`.

Для безопасного взаимодействия с защищенными данными используются специальные методы(геттеры и сеттеры) или декоратор `@property`.

```py
class User:
    def __init__(self, name, age):
        self.name = name        # Публичный атрибут
        self._age = age         # Защищенный атрибут
        self.__id = 12345       # Приватный атрибут

    @property
    def age(self):              # getter (чтение)
        return self._age

    @age.setter
    def age(self, value):       # setter (запись с проверкой)
        if value > 0:
            self._age = value
        else:
            print("Error: the age must be positive!")

user = User("Jack", 43)

print(user.age)
print(user._User__id) # доступ через Name Mangling (не рекомендуется)
user.age = -5

43
43
Error: the age must be positive!
```

**Использование инкапсуляции предлагает следующие возможности:**

- **Контроль данных** — обеспечивает возможность проверять входящие значения перед их сохранением (валидация).
- **Гибкость** — позволяет изменить внутреннюю логику хранения данных, не меняя внешний интерфейс.
- **Безопасность** — снижается риск случайного повреждения состояния объекта извне.

## Полиморфизм

**Полиморфизм** — это принцип ООП, позволяющий использовать единый интерфейс для работы с объектами разных типов.

**Ключевые концепции:**

- **Единый интерфейс** — разные классы могут реализовывать метод с одним и тем же именем, но выполнять его по-разному.
- **Утиная типизация (Duck Typing)** — если объект "крякает как утка" (имеет нужный метод), Python не важно, к какому классу он принадлежит. Можно вызвать метод и он будте работать.
- **Переопределение методов (Method Overriding)** — дочерний класс предоставляет свою специфическую реализацию метода, который уже есть в родительском классе.

```py
class Notification:  # Родительский класс
    def __init__(self, sender, text):
        self.sender = sender
        self.text = text

    def send(self):
        # Базовая логика, общая для всех
        print(f"Sending a message from {self.sender}: {self.text}")

class SMSNotification(Notification):  # Дочерний класс
    def send(self):  # Переопределение метода родителя
        print(f"SMS gateway: The message '{self.text}' was successfully sent to the number {self.sender}")

regular_note = Notification("System", "The update is complete!")
sms_note = SMSNotification("+8888888", "Your code: 5544")

# Вызов одного и того же интерфейса дает разный результат
regular_note.send()
sms_note.send()

# regular_note
Sending a message from System: The update is complete!
# sms_note
SMS gateway: The message 'Your code: 5544' was successfully sent to the number +8888888
```

**Применение полиморфизма обеспечивает:**

- **Гибкость архитектуры** — можно легко добавлять новые типы объектов в программу, не меняя существующий код, который с ними работает.
- **Читаемость** — код становится чище, так как нет необходимости проверять типы данных через `if isinstance(obj, ClassName)`.
- **Масштабируемость** — позволяет строить сложные системы с заменяемыми компонентами.

# Композиция и Агрегация

**Композиция и Агрегация** — это способы формирования сложных объектов из более простых. Вместо того чтобы наследоваться от огромного класса со всеми функциями, мы создаем маленькие независимые классы и включаем их как поля в основной класс.
Это позволяет строить сложные системы, избегая запутанных цепочек наследования.

**В чем разница:**

- **Композиция** — объект создаёт и полностью контролирует вложенный объект.

- **Агрегация** — объект получает уже существующий объект извне.

**Пример реализации композиции:**

```py
class Engine:
    def __init__(self, horsepower):
        self.horsepower = horsepower

    def start(self):
        print(f"The engine with a power of {self.horsepower} hp is running")

class Car:
    def __init__(self, brand, hp):
        self.brand = brand
        # Композиция: создаем объект Engine ВНУТРИ объекта Car
        self.engine = Engine(hp)

    def drive(self):
        print(f"The car {self.brand} is ready to go.")
        self.engine.start()

my_car = Car("Tesla", 500)
my_car.drive()

# The car Tesla is ready to go.
# The engine with a power of 500 hp is running
```

**Пример реализации агрегации отличается только в одном:**

```py
engine = Engine(hp)

class Car:
    def __init__(self, brand, engine):
        self.brand = brand
        # Агрегация: используем уже существующий engine ВНУТРИ объекта Car
        self.engine = engine 
```

В примере выше вместо того чтобы делать класс `Car` наследником класса `Engine` (что было бы странно, ведь машина не является двигателем), мы делаем двигатель атрибутом машины.

**Это дает следующие преимущества:**

- **Гибкость** — можно легко заменить деталь. Например, в класс `Car` можно передать другой двигатель (электрический или бензиновый), не переписывая всю иерархию наследования;
- **Избегание хрупкого базового класса** — при наследовании изменение в родителе может случайно сломать 10 наследников. При композиции или агрегации классы остаются независимыми;
- **Читаемость** — код становится похож на конструктор LEGO. Сложный функционал собирается из простых, протестированных кусочков;

# Магические методы

**Магические методы (Double Underline)** — это специальные методы класса, которые начинаются и заканчиваются двумя подчеркиваниями. Они позволяют перегружать стандартное поведение объектов и интегрировать их с синтаксисом Python (операторы `+`, `-`, функции `len()`, `print()` и др.).

**Основные методы:**

- `__init__(self, ...)` — инициализатор объекта, вызывающийся сразу после создания экземпляра для установки начальных значений атрибутов;
- `__new__(cls, ...)` — реальный конструктор, вызывающийся для создания самого экземпляра класса;
- `__del__(self)` — деструктор, вызывающийся при удалении объекта <a href="advanced.md#garbage-collection">сборщиком мусора</a>;
- `__str__(self)` — возвращает понятную для пользователя строку. Вызывается функциями `print()` и `str()`;
- `__repr__(self)` — возвращает формальное представление объекта для разработчика. Вызывается при отладке;
- `__len__(self)` — возвращает числовое значение, обычно длину объекта. Вызывается функцей `len()`;
- `__iter__(self)` — возвращает итератор. Вызывается циклом `for ... in`;
- `__add__(self, other)` — сложение `+`;
- `__sub__(self, other)` — вычитание `-`;
- `__mul__(self, other)` — умножение `*`;
- `__truediv__(self, other)` — деление `/`;
- `__eq__(self, other)` — равенство `==`;
- `__lt__(self, other)` — меньше `<`;
- `__gt__(self, other)` — больше `>`.

**Пример реализации:**

```py
class Wallet:
    def __init__(self, currency, balance):
        self.currency = currency
        self.balance = balance

    def __str__(self):
        # Определяем, как объект будет выглядеть в print()
        return f"Wallet ({self.currency}): {self.balance}"

    def __add__(self, other):
        # Позволяем складывать два объекта через '+'
        if self.currency != other.currency:
            raise ValueError("You can't add different currencies.")
        return Wallet(self.currency, self.balance + other.balance)

    def __len__(self):
        # Позволяет вызывать len(wallet) — вернет целое число баланса
        return int(self.balance)

w1 = Wallet("USD", 50)
w2 = Wallet("USD", 100)

w3 = w1 + w2  # __add__
print(w3)     # __str__ 
(len(w3)) # __len__

Wallet (USD): 150
150
```

**Грамотное использование магических методов делает код:**

- **Интуитивным** — объекты ведут себя как встроенные типы данных.
- **Лаконичным** — вместо вызова методов вроде `obj.add_another_obj(other)` можно использовать операторы.
- **Совместимым** — объекты класса могут корректно работать со стандартными библиотеками Python.

# Продвинутые концпепции

## Абстрактные классы

**Абстрактный класс** — это класс-шаблон, который нельзя использовать для создания объектов напрямую. Он служит основой для других классов, определяя обязательный набор методов, которые должны реализовать его наследники.

В Python абстрактные классы создаются с помощью **модуля abc (Abstract Base Classes)**. Основной инструмент — **<a href="advanced.md#decorators">декоратор</a> `@abstractmethod`**, который помечает методы как обязательные для переопределения.

**Пример реализации:**

```py
from abc import ABC, abstractmethod

class Payment(ABC): # Наследуемся от ABC, чтобы сделать класс абстрактным
    def __init__(self, amount):
        self.amount = amount

    @abstractmethod
    def process_payment(self):
        # Этот метод обязан реализовать каждый наследник
        pass

    def print_receipt(self):
        # Обычные методы тоже могут быть в абстрактном классе
        print(f"Check for the amount of: {self.amount} USD.")

class CreditCardPayment(Payment):
    def process_payment(self):
        print(f"Debit of {self.amount} USD from a bank card...")

class CryptoPayment(Payment):
    def process_payment(self):
        print(f"Transfer {self.amount} USD through a blockchain wallet...")

card = CreditCardPayment(1500)
card.process_payment()
card.print_receipt()

crypto = CryptoPayment(5000)
crypto.process_payment()
```

В примере выше попытка создать объект класса `Payment()` напрямую вызовет **ошибку**:
`TypeError: Can't instantiate abstract class Payment`.

**Ключевые особенности:**

- **Гарантия реализации** — если в дочернем классе не переопределены все методы, помеченные `@abstractmethod`, Python выдаст ошибку при попытке создать объект этого дочернего класса;
- **Единый интерфейс** — все наследники гарантированно обладают одинаковым набором методов, что позволяет безопасно использовать их в полиморфных функциях.

## Классы данных

**Классы данных** — это специальный инструмент Python из **модуля dataclasses**, предназначенный для создания классов, основной целью которых является хранение состояния (данных), а не реализация сложной логики.

Декоратор `@dataclass` автоматически генерирует методы `__init__`, `__repr__` и `__eq__` на основе аннотаций типов, которые в обычном классе пришлось бы писать вручную.

**Реализация на примере:**

```py
from dataclasses import dataclass

@dataclass
class OrderItem:
    item_id: int      # Артикул : целое число
    name: str         # Название : строка
    price: float      # Цена : вещ. число
    quantity: int = 1 # Количество : значение по умолчанию 

# Автоматический конструктор (__init__)
item1 = OrderItem(101, "Smartphone", 59900.0, 1)
item2 = OrderItem(102, "Case", 990.0) # quantity возьмется по умолчанию (1)

# Автоматическое строковое представление (__repr__)
print(item1) 
# Вывод: OrderItem(item_id=101, name='Смартфон', price=59900.0, quantity=1)

# Автоматическое сравнение по значениям (__eq__)
item3 = OrderItem(102, "Case", 990.0)
print(item2 == item3) 
# Вывод: True (сравниваются данные внутри, а не адреса в памяти)
```

**Преимущества использования `dataclass`:**

- **Лаконичность** — вместо 5–7 строк кода для `__init__` мы пишем одну строку с названием поля и его типом;

- **Читаемость** — сразу видно, какие данные хранит класс, без лишних конструкций и строк кода.

- **Готовые методы сравнения** — по умолчанию обычные классы в Python сравниваются по адресу в памяти (даже если данные внутри одинаковые, `==` вернет `False`). `dataclass` сравнивает именно содержимое полей;

- **Типизация** — использование аннотаций (`str`, `int`) помогает IDE и линтерам подсказывать ошибки еще на этапе написания кода.

Классы данных стоит использовать, когда класс — это просто контейнер для данных. Если же классу нужно хранить сложное состояние, иметь много внутренних методов и управлять доступом к данным через инкапсуляцию, лучше использовать обычные классы.
