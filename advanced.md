# Code review

**Code review (ревью кода)** - это **проверка исходного кода** программы с целью **обнаружения и исправления ошибок**, которые остались незамеченными **в начальной фазе разработки**.

**В процессе могут быть исправлены:**

- фактические ошибки;
- производительность кода;
- читабельность кода и ошибки форматирования кода.

**Целью ревью кода является улучшение качества программного кода.**

# Алгоритмы сортировки

**Алгоритм сортировки** — это алгоритм упорядочивания элементов в структурах данных. Алгоритмы сортировки оцениваются по скорости выполнения и эффективности использования памяти.

**Сортировка на месте** ­— алгоритмы сортировки, не потребляющие дополнительной памяти.

**Основные алгоритмы сортировки:**

**Медленные:**

- Пузырьковая сортировка (Bubble sort);
- Сортировка выбором (Selection sort);
- Сортировка простыми вставками (Insertion sort).

**Быстрые:**

- Сортировка Шелла (Shell sort);
- Быстрая сортировка (Quick sort);
- Сортировка слиянием (Merge sort);
- Пирамидальная сортировка (Heap sort);
- Сортировка TimSort (используется в Java и Python).

**Адаптивные алгоритмы сортировки:**

- Сортировка подсчетом (Counting sort)
- Блочная сортировка (Bucket sort)
- Поразрядная сортировка (Radix sort)

Адаптивные алгоритмы учитывают особенности входных данных и работают быстрее в определенных случаях.

## Базовые алгоритмы

**Сортировка пузырьком:**

```py
l = [8, 4, 7, 3, 1]

d = len(l)

for i in range(d - 1):
    for j in range(d - 1 - i):
        if l[j] > l[j + 1]:
            l[j], l[j + 1] = l[j + 1], l[j]
        print(l)
```

В примере выше мы сортируем список `l` методом пузырька. Разберем основные этапы сортировки:

1. `d = len(l)` — для удобства мы создадим переменную `d` и присвоим ей значение длины нашего списка `len(l)`. Длина списка `l = [8, 4, 7, 3, 1]` равна 5.

2. `for i in range(d - 1):` — создадим цикл `for` с переменной-счетчиком `i`, в диапазоне функции `range(d - 1)`. То есть переменная `i` будет принимать значения от `0` до `4` невключительно(`0, 1, 2, 3`).

3. `for j in range(d - 1 - i):` — создадим вложенный цикл `for` с переменной-счетчиком `j`, в диапазоне функции `range(d - 1 - i)`. Значением `i` в первой итерации цикла равняется `0`, а значит `j` работает в диапазоне от `0` до `4` невключительно(`0, 1, 2, 3`).

4. `if l[j] > l[j + 1]:` — с помощью условной конструкции `if` проверяем значение списка `l` по индексу `j`(то есть `l[0] = 8`). Если значение `l[j]` больше, чем `l[j + 1]`(`l[1] = 4`), то условие истинно и мы переходим к выполнению тела условия.

5. `l[j], l[j + 1] = l[j + 1], l[j]` — на этом этапе мы с помощью [множественного присваивания](operators.md#операторы-присваивания) меняем местами значения в списке. То есть `l[j] = 8` теперь равно `j[j + 1] = 4`, а `l[j + 1] = 4` теперь равно `l[j] = 8`. Исходный список теперь выглядит так: `[4, 8, 7, 3, 1]`.

6. Далее мы переходим к следующей итерации вложенного цикла и выполняем этапы 3, 4 и 5 снова. С каждой итерацией сравниваются пары значений списка и большее значение двигается в правую сторону. То есть на выходе из вложенного цилка список будет выглядеть так: `[4, 7, 3, 1, 8]`. Как видно элемент `8` занял свою позицию в списке. Внешний цикл с переменной `i` переходит к своей следующей итерации.

7. Когда внешний цикл переходит к своей следующей итерации, увеличивается значение переменной `i`. И соотвественно диапазон `d - 1 - i` вложенного цикла с переменной `j` уменьшается. Таким образом мы откидываем отсортированную часть списка и наш цикл работает только с неотсортированными значениями. То есть `[4, 7, 3, 1]`, элемент `[8]` принадлежит отсортированной части списка, и в сортировке больше не принимает участия.

8. Таким образом сортировка проходит по каждому этапу. Для наглядности, вот как выглядит исходный список на каждом этапе сравнения пар элементов.

    ```py
    [8, 4, 7, 3, 1] # исходный список
    # Первая итерация внешнего цикла (i = 0)
    [4, 8, 7, 3, 1] # 8 > 4
    [4, 7, 8, 3, 1] # 8 > 7
    [4, 7, 3, 8, 1] # 8 > 3
    [4, 7, 3, 1, 8] # 8 > 1
    # Вторая итерация внешнего цикла (i = 1)
    [4, 7, 3, 1, 8] # 4 < 7
    [4, 3, 7, 1, 8] # 7 > 3
    [4, 3, 1, 7, 8] # 7 > 1
    # Третья итерация внешнего цикла (i = 2)
    [3, 4, 1, 7, 8] # 3 < 4 
    [3, 1, 4, 7, 8] # 4 > 1
    # Четвертая итерация внешнего цикла (i = 3)
    [1, 3, 4, 7, 8] # 1 < 3
    ```

Сортировку пузырьком можно оптимизировать. Как мы видели в примере выше каждую итерацию внешнего цикла значения в списке менялись местами. Но что если представить, что наш список `l` изначально выглядел таким образом: `[8, 1, 3, 4, 7]`. Как вы можете заметить в этом списке есть подсписок уже осортированных значений `[1, 3, 4, 7]`. То есть большая часть списка уже отсортированы. Алгоритму нужно только разместить элемент `8` в конец списка и список будет полностью отсортирован. Но текущий алгоритм после размещения элемента `8` в конец списка будет дальше сравнивать значения в списке, даже если он уже полностью отсортирован.

Попробуем посмотреть каждый этап сортировки такого списка текущим алгоритмом.

```py
[8, 1, 3, 4, 7] # исходный список
# Первая итерация внешнего цикла (i = 0)
[1, 8, 3, 4, 7] # 8 > 1
[1, 3, 8, 4, 7] # 8 > 3
[1, 3, 4, 8, 7] # 8 > 4
[1, 3, 4, 7, 8] # 8 > 7
# Вторая итерация внешнего цикла (i = 1)
[1, 3, 4, 7, 8] # 1 < 3
[1, 3, 4, 7, 8] # 3 < 4
[1, 3, 4, 7, 8] # 4 < 7
# Третья итерация внешнего цикла (i = 2)
[1, 3, 4, 7, 8] # 1 < 3
[1, 3, 4, 7, 8] # 3 < 4
# Четвертая итерация внешнего цикла (i = 3)
[1, 3, 4, 7, 8] # 1 < 3
```

Как вы можете видеть внешний цикл совершает еще три итерации и сравнивает значения, хотя список уже полностью отсортирован. В таком случае будет логичным доработать алгоритм, чтобы после полной отсортировки алгоритм останавливался и выдавал результат.

Сделать это можно с помощью перменной флага `isDone`:

```py
l = [8, 1, 3, 4, 7]

d = len(l)

for i in range(d - 1):
    isDone = True
    for j in range(d - 1 - i):
        if l[j] > l[j + 1]:
            l[j], l[j + 1] = l[j + 1], l[j]
            isDone = False
    if isDone:
        break
print(l)
```

В примере выше **при каждой новой итерации внешнего цикла переменной `isDone` присваивается значение `True`**. Если во время выполнения итерации, во вложенном цикле выполнится условие, то переменная **примет значение `False`**. То есть, если итерация внешнего цикла заканчивается без каких либо изменений в списке(то есть список уже полностью отсортирован), то значение переменной останется `True`, и когда она будет проходить конструкцию `if isDone:`, условие будет истинным и цикл остановится командой `break`. Таким образом мы можем избежать выполнения лишних итераций в цикле и пустых проверок.

```py
[8, 1, 3, 4, 7] # исходный список
# Первая итерация внешнего цикла (i = 0)
[1, 8, 3, 4, 7] # 8 > 1
[1, 3, 8, 4, 7] # 8 > 3
[1, 3, 4, 8, 7] # 8 > 4
[1, 3, 4, 7, 8] # 8 > 7
# Цикл продложает работу, так как в списке были изменения и переменная isDone = False
# Вторая итерация внешнего цикла (i = 1)
[1, 3, 4, 7, 8] # 1 < 3
[1, 3, 4, 7, 8] # 3 < 4
[1, 3, 4, 7, 8] # 4 < 7
# Цикл прерывается, так как изменений в списке не последовало и значением переменной isDone осталось True.
```

**Примечания:**

- Алгоритм сортировки пузырьком считается учебным и практически не применяется вне учебной литературы, а на практике применяются более эффективные.

**Сортировка выбором:**

```py
l = [8, 4, 7, 3, 1]

d = len(l)

for i in range(d - 1):
    mx = d - 1 - i
    for j in range(d - 1 - i):
        if l[j] > l[mx]:
            mx = j
    l[d - 1 - i], l[mx] = l[mx], l[d - 1 - i]

print(l)
```

В примере выше мы сортируем список `l` сортировкой выбором. Разберем основные этапы сортировки:

1. `d = len(l)` — для удобства мы создадим переменную `d` и присвоим ей значение длины нашего списка `len(l)`. Длина списка `l = [8, 4, 7, 3, 1]` равна 5.

2. `for i in range(d - 1):` — создадим цикл `for` с переменной-счетчиком `i`, в диапазоне функции `range(d - 1)`. То есть переменная `i` будет принимать значения от `0` до `4` невключительно(`0, 1, 2, 3`).

3. В теле внешнего цикла каждую итерацию мы будем присваивать переменной `mx` значение `d - 1 - i`. В первой итерации это значение `5 - 1 - 0` равно `4`, так как `i = 0`. Таким образом каждую итерацию переменная `mx` будет содержать индекс последнего элемента неотсортированной части списка(в нашем случае этот элемент `l[4] = 1`).

4. `for j in range(d - 1 - i):` — создадим вложенный цикл `for` с переменной-счетчиком `j`, в диапазоне функции `range(d - 1 - i)`. Значением `i` в первой итерации цикла равняется `0`, а значит `j` работает в диапазоне от `0` до `4` невключительно(`0, 1, 2, 3`). Обратим внимание, что последний элемент неотсортированной части списка не попадает в этот диапазон. Он уже присвоен переменной `mx`, а сравнивать его с самим собой это одна лишняя итерация вложенного цикла.

5. `if l[j] > l[mx]:` — с помощью условной конструкции `if` проверяем значение списка `l` по индексу `j`(`l[0] = 8`). Если значение `l[j]` больше, чем `l[mx]`(`l[4] = 1`), то условие истинно и мы переходим к выполнению тела условия.

6. `mx = j` — в теле условия мы обновляем значение переменной `mx`, поскольку `mx` должна хранить индекс максимального элемента списка.

7. Далее мы переходим к следующей итерации вложенного цикла и выполняем этапы 4, 5 и 6 снова. С каждой итерацией сравнивается значение `l[mx]` и `l[j]`. В случае, если мы находим значение больше чем `l[mx]` мы обновляем переменную `mx`, записывая в нее новый индекс максимального элемента в списке. В нашем списке `[8, 4, 7, 3, 1]` нет элемента больше, чем `8`, поэтому переменная `mx` по окончанию вложенного цикла будет иметь значение `0`.

8. `l[d - 1 - i], l[mx] = l[mx], l[d - 1 - i]` — после выхода из вложенного цикла с помощью множественного присваивания, мы присваиваем последнему элементу неотсортированного списка — `l[d - 1 - i]`, значение максимального элемента списка — `l[mx]`, а `l[mx]` получает значение `l[d - 1 - i]`. То есть в нашем случае `l[4] = l[0]`, а `l[0] = l[4]`. Таким образом в конце первой итерации внешнего цикла исходных список будет иметь вид: `[1, 4, 7, 3, 8]`. Алгоритм нашел максимальный элемент `8` и разместил его в конце списка.

9. Когда внешний цикл переходит к своей следующей итерации, увеличивается значение переменной `i`. И соотвественно диапазон `d - 1 - i` вложенного цикла с переменной `j` уменьшается. Таким образом мы откидываем отсортированную часть списка и наш цикл работает только с неотсортированными значениями. То есть `[1, 4, 7, 3]`, элемент `[8]` принадлежит отсортированной части списка, и в сортировке больше не принимает участия.

10. Таким образом сортировка проходит по каждому этапу. Цель каждой итерации внешнего цикла: найти максимальный элемент неотсортированной части списка и разместить его в конец неотсортированной части. Для наглядности вот как выглядит исходный список в конце каждой итерации внешнего цикла.

    ```py
    [8, 4, 7, 3, 1] # исходный список
    # Первая итерация внешнего цикла (i = 0)
    [1, 4, 7, 3, 8] # l[mx] = 8
    # Вторая итерация внешнего цикла (i = 1)
    [1, 4, 3, 7, 8] # l[mx] = 7
    # Третья итерация внешнего цикла (i = 2)
    [1, 3, 4, 7, 8] # l[mx] = 4
    # Четвертая итерация внешнего цикла (i = 3)
    [1, 3, 4, 7, 8] # l[mx] = 3
    ```

**Примечания:**

- Алгоритм сортировки выбором также считается учебным и практически не применяется вне учебной литературы. На практике используют более эффективные алгоритмы.

**Сортировка простыми вставками:**

```py
l = [8, 10, 7, 3, 1]

d = len(l)

for i in range(1, d): 
    elem = l[i]
    j = i

    while j >= 1 and l[j - 1] > elem:
        l[j] = l[j - 1]
        j -= 1
    l[j] = elem

print(l)
```

В примере выше мы сортируем список `l` сортировкой простыми вставками. Разберем основные этапы сортировки:

1. `d = len(l)` — для удобства мы создадим переменную `d` и присвоим ей значение длины нашего списка `len(l)`. Длина списка `l = [8, 10, 7, 3, 1]` равна 5.

2. `for i in range(d - 1):` — создадим цикл `for` с переменной-счетчиком `i`, в диапазоне функции `range(1, d)`. То есть переменная `i` будет принимать значения от `1` до `5` невключительно(`1, 2, 3, 4`). Таким образом мы разделяем наш список на две части. Первая часть — `l[0:1] = [8]`, и вторая часть — `l[1:] = [10, 7, 3, 1]`.

3. `elem = l[i]` — созадем переменную `elem` и присваиваем ей значение `l[i]`(`l[1] = 10`). Переменная `elem`, каждую итерацию внешнего цикла, хранит первый элемент неотсортированной части списка.

4. `j = i` — создаем переменную `j` — счетчик вложенного цикла и присваиваем ей значение переменной `i`(`j = 1`).

5. `while j >= 1 and l[j - 1] > elem:` — создаем цикл `while` с условием: пока переменная `j` больше либо равна `1`, и значение списка `l` по индексу `j - 1` больше, чем переменная `elem`, выполняется тело цикла. В нашей первой итерации цикла значение переменной `elem`(`10`) больше, чем значение `8` которое уже находится у нас в отсортированной части списка, а значит условие ложно и мы пропускаем тело цикла и вернемся к нему во второй итерации внешнего цикла `for`.

6. `l[j] = elem` — как мы помним переменная `elem` хранит значение элемента списка по индексу `l[i]`(`10`). То есть в конце первой итерации мы присваиваем `l[j]` значение `l[i]`. Что есть `l[1]` = `l[1]`. То есть элемент не меняет своей позиции и список остается неизменным(`[8, 10, 7, 3, 1]`).

7. Переходим ко второй итерации внешнего цикла. Теперь значение переменной `i = 2`, а значит `elem = l[2]`(`7`). Переменная `j = i` и равна `2`. Наш список снова делится на 2 части. Первая часть `[8, 10]` — **отсортированная**(все элементы стоят по возрастанию), вторая часть `[7, 3, 1]` — **неотсортированная**. Теперь перейдем к вложенному циклу `while`. Переменная `j = 2` и проходит первое условие цикла (`j >= 1`). Значение списка `l` по индексу `j - 1`, то есть `10`(`l[1]`), больше, чем значение `elem` равное `7`, а значит оба условия верны и выполняется тело цикла.

8. `l[j] = l[j - 1]` — на этом этапе мы присваиваем `l[j]`(`l[2]`) значение под индексом `l[j - 1]`(`l[1]`). То есть в нашем случае `l[2] = 10` и список преобретает вид `[8, 10, 10, 3, 1]`. Обратите внимание, что мы клонировали элемент `10` на один индекс вправо, но при этом не потеряли значение `7`, ведь оно хранится у нас в переменной `elem`.

9. `j -= 1` — отнимаем 1 от переменной `j`, чтобы перейти к следующему элементу в нашей отсортированной части списка `[8, 10]`. В нашем случае `j - 1 = 1`. `l[j - 1]`(`l[0] = 8`), больше чем `elem = 7`, поэтому мы переходим к следующей итерации цикла `while`. В этой итерации мы клонируем уже элемент `l[0] = 8` вправо на один индекс и список преобретает вид [`8, 8, 10, 3, 1`]. Снова отнимаем от переменной `j -= 1`  и переходим к следующей итерации цикла.

10. Переменная `j` не проходит условие цикла так как `j - 1 = 0`, а по условию цилка `j >= 1` значение `j` должно быть больше, либо равно `1`.  Цикл завершается и мы переходим к концу второй итерации внешнего цикла `for` и присваиваем `l[j]`(`l[0]`) значение переменной `elem`(`7`). То есть на выходе из второй итерации внешнего цикла список преобретает вид `[7, 8, 10, 3, 1]`.

11. Далее мы продолжаем двигаться по списку и перемещать элементы из неотсортированной части в отсортированную.

12. Как вы могли заметить, смысл алгоритма в том, чтобы взять переменную из неотсортированной части списка и найти ей место в уже отсортированной части. Поскольку мы знаем, что все элементы отсортированной части уже стоят в правильном порядке, то просто двигаем `elem` пока не найдется элемент меньший чем он. После того как мы нашли этот элемент, нам нет необходимости сравнивать `elem`, со всеми остальными элементами отсортированной части, потому что мы уже знаем, что элементы далее будут меньше.

---

Алгоритм сортировки простыми вставками делает меньше сравнений, чем алгоритм сортировки пузырьком или выбором. Более того мы можем оптимизировать этот алгоритм, чтобы сократить количество сравнений еще больше. Сделать это можно с помощью **бинарного поиска**.

**Бинарный поиск** — это **эффективный алгоритм поиска позиции заданного элемента** (или места для его вставки) **в отсортированной** последовательности, основанный на последовательном **делении области поиска пополам**. На каждом шаге алгоритм сравнивает искомое значение со с**редним элементом текущего диапазона и отбрасывает половину элементов**, в которой искомый элемент **заведомо отсутствовать не может**.  

```py
left, right = 0, i

while left < right:
    mid = (left + right) // 2
    if l[mid] > elem:
        right = mid
    else:
        left = mid + 1
```

В примере выше переменна `i` — это индекс элемента в массиве от которого отталкивается поиск. Переменные `left` и `right` это границы диапазона поиска нужного индекса. Цикл продолжается пока граница `left` меньше границы `right`. При этом каждую итерацию алгоритм находит среднее значение `mid` в диапазоне `[left:right]` и если `l[mid]` больше, чем `elem`, то граница диапазона `right` смещается до `mid`. Если же `l[mid]` меньше, либо равен `elem`, левая граница смещается к `mid + 1`. Таким образом, по окончанию алгоритма в переменной `left` хранится значение искомого индекса.

Вот реализация алгоритма сортировки простыми вставками с использованием бинарного поиска:

```py
l = [8, 4, 7, 3, 1]

d = len(l)

for i in range(1, d): 
    elem = l[i]
    j = i

    if j >= 1 and l[j - 1] > elem:
        left, right = 0, i

        while left < right:
            mid = (left + right) // 2
            if l[mid] > elem:
                right = mid
            else:
                left = mid + 1

        while i > left:
            l[i] = l[i - 1]
            i -= 1

        l[left] = elem
        
print(l)
```

Как вы могли заметить начало алгоритма полностью совпадает с оригиналом, но способ поиска нужного индекса для вставки `elem` в список, реализован с помощью бинарного поиска, а смещение элементов цикла в правую сторону вынесено в отдельный цикл. Таким образом мы получаем тот же алгоритм, но работающий гораздо быстрее оригинального.
