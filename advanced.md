# Итерируемые объекты и итераторы

## Что такое итерируемый объект?

**Итерируемый объект** — это объект в языке Python, который реализует [специальный метод](oop.md#магические-методы) `__iter__`, возвращающий итератор, либо метод `__getitem__` с поддержкой целочисленной индексации, позволяющий последовательно получать свои элементы.

В языке Python такими объектами являются списки, словари, кортежи и т.д. Помимо этого мы можем создавать свои объекты, реализующие метод `__iter__`, тем самым делая их итерируемыми.

**Разберем на примере сундука с предметами из какой нибудь игры:**

```py
class Chest: # класс сундука
    def __init__(self, items):
        self.items = items

    def add_item(self, item):
        self.items.append(item)

    def __iter__(self): # специальный метод __iter__
        return iter(self.items) # с помощью встр. функции iter создаем итератор


class Item: # класс предмета
    def __init__(self, name, price):
        self.name = name
        self.price = price

    def __str__(self):
        return f"{self.name} - {self.price} coins" # красивый вывод для предмета
```

Создаем сундук и проходим по предметам в сундуке через цикл `for`:

```py
sword = Item("Sword", 10) # создаем предметы
health_pot = Item("Health Potion", 2)
gold = Item("Gold", 12)

chest = Chest([sword, health_pot, gold]) # кладем их в сундук

shield = Item("Shield", 5)

chest.add_item(shield) # добавляем еще один предмет в сундук

for item in chest: # итерируем объект
    print(item)

# Sword - 10 coins
# Health Potion - 2 coins
# Gold - 12 coins
# Shield - 5 coins
```

Когда в программе вызывается цикл `for`, Python вызывает у объекта `chest` его специальный метод `__iter__`. Если его в объекте нет, он пробует вызывать метод `__getitem__`, чтобы через него создать объект-итератор. Если у объекта нет ни одного из этих двух методов, выбрасывается исключение `TypeError: 'Chest' object is not iterable`. В нашем случае метод `__iter__` есть и он возвращает объект итератор. Объект итератор, полученный из метода `__iter__` реализует специальный метод `__next__`, возвращая каждую итерацию следующее значение.

## Что такое итератор?

**Итераторы** — это объекты в Python, которые реализуют **протокол итерации**, то есть умеют **последовательно возвращать элементы из некоторой коллекции или последовательности**.

```py
class Counter:
    def __init__(self, start, end):
        self.current = start
        self.end = end

    def __iter__(self): # специальный метод __iter__
        return self

    def __next__(self): # специальный метод __next__
        if self.current > self.end: 
            raise StopIteration

        value = self.current
        self.current += 1

        return value
```

Класс Counter, в примере выше, это самописный итератор. При инициализации он получает начальный и конечный индексы.

Он реализует специальный метод `__next__`, который:

- проверяет индекс : `if self.current > self.end`;
- если текущий индекс меньше конечного, продолжает выполнение, иначе останавливает итерацию: `raise StopIteration`;
- определяет переменную `value`: `value = self.current`;
- увеличивает текущий индекс на единицу: `self.current += 1`;
- возвращает значение `value`;

Чтобы лучше понять, как работает итератор, рассмотрим вызов цикла `for` в программе:

```py
obj = Counter(1, 3)

for i in obj: # obj.__iter__()
    print(i, end=" ") # 1 2 3
```

В примере выше, как и в случае с итерируемым объектом `chest`, Python сначала обращается к методу объекта `__iter__`. У итератора `Counter` есть метод `__iter__`, который возвращает `self`. Итератор обязан иметь `__iter__`, возвращающий самого себя. Это сделано для того, чтобы его можно было использовать там же, где и итерируемый объект. Далее у итератора вызывается метод `next`, работу которого мы разбирали выше.

Итераторы - одноразовые. После единичного использования они становятся пустыми и последующие их вызовы ничего не возвращают.

Так же мы можем получить значения из итератора с помощью функции next():

```py
obj = Counter(1, 3)

print(next(obj)) # 1
print(next(obj)) # 2
```

Метод `__iter__` у итерируемого объекта создает и возвращает новый объект-итератор. Его можно перебирать сколько угодно раз — каждый раз создается новый итератор. Итератор же можно перебрать только один раз и его метод `__iter__` возвращает `self`.

**Особенности итераторов:**

- **Сохранение состояния** — итераторы сохраняют своё состояние между вызовами, что позволяет им продолжить выполнение с места, где они были прерваны;
- **Ленивые вычисления** — итераторы производят значения только "по требованию", что экономит память. Это особенно полезно при работе с большими данными или бесконечными последовательностями. Однако ошибки внутри функции могут появиться в середине вашего цикла for, а не сразу при вызове функции;
- **Одноразовость** — Итератор можно перебрать только один раз.

**Примечания:**

- итераторы можно получить, как результат работы некоторых встроенных функций и методов (напимер `dict.items`, `map`, `filter`)
- итератор можно преобразовать в список или кортеж, с помощью функций `list()` и `tuple()`;
- итератор можно распаковать с помощью оператора распаковки `*`.

# Генераторы

## Основные понятия

**Генератор** — это специализированный тип итератора, позволяющий реализовать протокол итерации в лаконичной форме. Вместо описания класса с методами `__iter__` и `__next__`, используется функция с ключевым словом `yield`.

```py
def counter_generator(start, end):
    current = start
    while current <= end:
        yield current  # Приостановка и возврат значения
        current += 1

gen = counter_generator(1, 3)
print(next(gen)) # 1
print(next(gen)) # 2
print(next(gen)) # 3
# Следующий вызов next(gen) инициирует StopIteration.
```

**Механизм работы генератора:**

- Вызов функции не исполняет код сразу, а возвращает объект-генератор.
- Оператор `yield` используется внутри функции для превращения её в генератор.
 Он приостанавливает выполнение функции, сохраняя её состояние (локальные переменные и указатель инструкций), и возвращает значение.
- Исключение `StopIteration` вызывается автоматически, когда код функции доходит до конца.

Генераторы имеют все особенности итераторов, при этом обеспечивая лаконичность кода. В генераторах отсутствует необходимость вручную управлять состоянием объекта (например, `self.current`) и явно возбуждать исключения.

Как и итераторы, генераторы одноразовые, что означает — после единичного использования они становятся пустыми и последующие их вызовы ничего не возвращают.

## Генераторные выражения

Для упрощенной логики итерации в Python используются **генераторные выражения (generator expressions)**. Они представляют собой компактный однострочный синтаксис, позволяющий создать объект-генератор без объявления именованной функции.

```py
squares = (x**2 for x in range(1000000))

print(next(squares)) # 0
print(next(squares)) # 1
```

Генераторные выражения оптимальны для создания простых итераторов с минимальными затратами памяти.

Синтаксически они идентичны [генераторам списков(list comprehensions)](lists.md#генераторы-списков), [генераторам множеств(set comprehensions)](sets.md#генераторы-множеств) или [генераторам словарей(dictionary comprehensions)](dicts.md#генераторы-словарей).

Это неудивительно, ведь они послужили основой для списковых, множественных и словарных генераторов. Но у них есть одно ключевое различие: генераторное выражение **возвращает объект-итератор**, в то время как остальные конструкции возвращают **готовый контейнер**, заполненный данными.

# Обработка ошибок и исключений

## Ошибки в Python

**Есть 2 типа ошибок:**

- **Синтаксические ошибки (syntax error)** — появляются в результате нарушения синтаксиса языка программирования при написании исходного кода.
- **Ошибки выполнения (runtime error)** — появляются в процессе выполнения программы. Подобные ошибки еще называются **исключениями**.

В Python есть следующие базовые типы исключений. От этих классов наследуются все конкретные типы исключений.

- **BaseException**: базовый тип для всех встроенных исключений;

- **Exception**: базовый тип, который обычно применяется для создания своих типов исключений;

- **ArithmeticError**: базовый тип для исключений, связанных с арифметическими операциями (`OverflowError`, `ZeroDivisionError`, `FloatingPointError`);

- **BufferError**: тип исключения, которое возникает при невозможности выполнить операцию с буффером;

- **LookupError**: базовый тип для исключений, которое возникают при обращении в коллекциях по некорректному ключу или индексу (например, `IndexError`, `KeyError`).
  
## Обработка ошибок

Конструкция обработки ошибок может иметь следующий вид:

```py
try:
    инструкции
except [Тип_исключения]:
    инструкции
else:
    инструкции
finally:
    инструкции
```

Весь основной код, в котором потенциально может возникнуть исключение, помещается после ключевого слова `try`. Если в этом коде генерируется исключение, то работа кода в блоке `try` прерывается, и выполнение переходит в блок `except`.

При обработке исключений также можно использовать необязательные блоки `else` и `finally`. Блок `else` выполняется, если в блоке `try` не произошло исключение. Блок `finally` выполняется вне зависимости, было ли сгенерировано исключение.

Чтобы обрабатывать несколько типо исключений необходимо исользовать кортеж в блоке except:

```py
try:
    инструкции
except ([Тип_исключения], [Тип_исключения]):
    инструкции
```

Можно получить объект исключения с помощью конструкции `except SomeError as e`.

Основные свойства:

- `e.args` — кортеж аргументов, переданных при создании исключения.
- `str(e)` — человекочитаемое описание ошибки.
- `e.__class__` — тип исключения.
- `e.__traceback__` — объект traceback (информация о стеке вызовов).
- `e.__cause__` — исходное исключение (если использовался raise ... from).
- `e.__context__` — предыдущее исключение, если оно было.

Пример:

```py
try:
    1 / 0
except ZeroDivisionError as e:
    print(e.args) # ('division by zero',)
    print(str(e)) # division by zero
    print(e.__class__) # <class 'ZeroDivisionError'>

```

## Кастомные исключения и оператор raise

В языке Python мы не ограничены только встроенными типами исключений и можем, применяя наследование, при необходимости создавать свои типы исключений.

```py
class PersonAgeException(Exception):
    def __init__(self, age, minage, maxage):
        self.age = age
        self.minage = minage
        self.maxage = maxage
 
    def __str__(self):
        return f"Недопустимое значение: {self.age}. " \
               f"Возраст должен быть в диапазоне от {self.minage} до {self.maxage}"
```

**Оператор raise** — позволяет вручную генерировать исключения.

Также с помощью конструкции `raise NewError() from e` можно создать новое исключение и связать его с исходным (exception chaining).

```py
try:
    int("abc")
except ValueError as e:
    raise RuntimeError("Ошибка обработки данных") from e
```

В traceback будет:

- исходная ошибка (`ValueError`);
- новая ошибка (`RuntimeError`);
- указание, что одна вызвана другой.

# Работа с файлами

## Базовые понятия

**Файл ввода** — файл, из которого программа читает данные.

**Файл вывода** — файл, куда сохраняются данные.

**Способы получения доступа к данным в файле:**

- **Последовательный** — выдает порции информации одну за другой,  непозволяя переходить сразу к нужной части данных (сначала придется прочитать все предыдущие).
- **Прямой или произвольный** — позволяет переходить непосредственно к любой порции данных, не читая предыдущие.

## Открытие и закрытие файла

**Функция `open(path, mode, encoding)`** — создает файловый объект (дескриптор файла) и связывает его с файлом на диске.

**Синтаксис:**

`файловая_переменная = open(путь_к_файлу, режим_доступа, кодировка)`

где `файловая переменная` – имя переменной, которая ссылается на файловый объект, `путь_к_файлу(path)` – строковый литерал, задающий путь к файлу, `режим_доступа(mode)` – строковый литерал, задающий режим доступа, в котором файл будет открыт (по умолчанию `r`), `кодировка(encoding)` — строковый литерал, задающий кодировку файла.

**Режимы доступа к файлу:**

| Режим | Название            | Описание                                                                                                  |
|-------|---------------------|-----------------------------------------------------------------------------------------------------------|
| `'r'` | Read (чтение)       | Открыть файл только для чтения. Такой файл не может быть изменён.                                         |
| `'w'` | Write (запись)      | Открыть файл для записи. Если файл не существует, создает новый. Если существует, стирает его содержимое. |
| `'a'` | Append (добавление) | Открыть файл для записи. Данные будут добавлены в конец файла. Если файл не существует, он будет создан.  |
| `'r+'`| Read + Write        | Открыть файл для чтения и записи. В этом режиме происходит частичная перезапись содержимого файла.        |
| `'x'` | Create (создание)   | Создать новый файл. Если файл существует, произойдёт ошибка.                                              |

Строковые литералы, которые можно использовать для задания режима обработки файла указываются после режима доступа к файлу (`rt`, где `r` — чтение, `t` — текстовый режим).
Помимо текстового режима обработки файлов `t` есть также бинарный режим `b`, для работы с бинарными файлами (картинки, звук и т.д.).

После того как файл открыт, мы можем получить информацию о нем, через свойства:

| Атрибут (свойство) | Описание                                                      |
|--------------------|---------------------------------------------------------------|
| `my_file.closed`   | Возвращает `True`, если файл закрыт, иначе — `False`.         |
| `my_file.mode`     | Возвращает режим доступа, с помощью которого был открыт файл. |
| `my_file.name`     | Возвращает имя файла.                                         |
| `my_file.encoding` | Возвращает кодировку открытого файла.                         |

```py
my_file = open('my_file.txt', 'r', encoding='UTF-8')

print(my_file.name, my_file.mode, my_file.encoding, my_file.closed)

my_file.txt r UTF-8 False
```

---

Файловый метод `close()` используется для закрытия файла.

```py
my_file = open('my_file.txt', 'r', encoding='UTF-8') # открываем файл

# обрабатываем данные

my_file.close() # закрываем файл
```

После окончания работы с файлом его **необходимо закрыть**.  
Для этого есть несколько причин:

- если файл изменялся, это позволит корректно его сохранить;
- если открытый файл потребуется другим программам, ваша программа может его блокировать;
- не стоит держать в памяти лишние, уже не нужные данные.

В некоторых операционных системах **невыполнение операции закрытия** файла может привести к **потере данных**. Данные сначала пишутся в **буфер** – небольшую область временного хранения в оперативной памяти. Когда буфер заполняется, система записывает его содержимое в файл. Процесс закрытия файла записывает любые несохраненные данные из буфера в файл. Чтобы **принудительно** записать содержимое буфера в файл, используется файловый метод `flush()`.

---

Хочется иметь возможность **автоматически закрывать файл** сразу после окончания работы с ним и осуществлять закрытие даже при возникновении ошибки. Файловые дескрипторы уже умеют работать в таком режиме, но для этого их нужно использовать как **менеджеры контекста.**

## Менеджер контекста

**Менеджер контекста** — объект, реализующий одноименный протокол, позволяющий использовать следующий специальный синтаксис:

```py
with object as name:
    # Здесь нам доступен ресурс name.
    # Это тело with-блока.
# А здесь ресурс name уже освобождён, даже если в теле with-блока произошла ошибка.
```

С помощью него мы можем работать с файлами:

```py
with open('my_file.txt') as file:
    # обработка файла
```

По выходу из контекста ресурс **автоматически освобождается, даже если при выполнении блока возникло исключение**.

Менеджера контекста позволяет работать с несколькими файлами:

```py
with open('input.txt', 'r') as input_file, open('output.txt', 'w') as output_file:
    # обработка файлов
```

Конструкция `with - as` применяется для получения гарантии того, что критические функции выполнятся в любом случае.

**Выполнение конструкции `with - as` можно представить следующим образом:**

1. Выполняется выражение в конструкции `with - as`;
2. Загружается [специальный метод](oop.md#магические-методы) `__exit__` для дальнейшего использования;
3. Выполняется метод `__enter__`, и если конструкция **with** включает в себя слово **as**, то возвращаемое методом `__enter__` значение записывается в переменную;
4. Выполняются инструкции блока кода;
5. Вызывается метод `__exit__`, причём неважно, выполнилось ли тело конструкции или произошло исключение. В этот метод передаются параметры исключения, если оно произошло, или во всех аргументах значение `None`, если исключения не было.

При работе с файлами желательно всегда использовать менеджер контекста. Это делает программу **надежнее**.

## Чтение данных из файла

У нас есть файл `input.txt` со следующим содержимым:

```txt
Python
Java
C++
C#
JavaScript
Ruby
```

Для считывания данных из этого файла используем следующие методы: `f.read()`, `f.readline()`, `f.readlines()`.

**Файловый метод `file.read()`** — считывает все содержимое из файла и возвращает строку, которая может содержать символы перехода на новую строку `\n` и другие (`\t`, `\r`).

```py
with open('input.txt') as file:
    data = file.read() # 'Python\nJava\nC++\nC#\nJavaScript\nRuby'
```

---

**Файловый метод `file.readline()`** — считывает одну строку из файла (до символа конца строки `\n`), при этом возвращается считанная строка вместе с символом `\n`. Если считать строку не удалось – достигнут конец файла и больше строк в нем нет, возвращается пустая строка.

```py
with open('input.txt') as file:
    line = file.readline() # 'Python\n'
```

Также мы можем перебрать построчно содержимое файла с помощью цикла `for`:

```py
with open('input.txt') as file:
    for line in file:
        print(line, end='')

Python
Java
C++
C#
JavaScript
Ruby
```

---

**Файловый метод `file.readlines()`** — считывает все строки из файла и возвращает список из всех считанных строк (одна строка — один элемент списка). При этом, каждая строка в списке заканчивается символом переноса строки  `\n`.

```py
with open('input.txt') as file:
    data = file.readlines() # ['Python\n', 'Java\n', 'C++\n', 'C#\n', 'JavaScript\n', 'Ruby']
```

## Запись данных в файл

Существует несколько способов записи данных в файл.

**Файловый метод `file.write(str)`** — записывает переданную строку `str` в файл.

```py
with open('output.txt', 'w') as output:
    output.write('Hello World!')
```

---

**Файловый метод `file.writelines(lst)`** — записывает переданный список строк в файл.

```py
with open('output.txt', 'w') as output:
    lst = ['Hello', 'World', '!']
    output.write(lst)
```

файловые методы `file.write()` и `file.writelines()` не добавляют переход на новую строку, поэтому для перехода на новую строку в файле необходимо явно добавить символ `\n`.

---

**Функция `print(str, file=None)`** — имеет необязательный параметр `file`, позволяющий указать файл для записи.

```py
with open('output.txt', 'w') as output:
    print('Hello World!')
```

## Курсор в файле

При открытии файла текущая позиция всегда равна **нулю** – указывает на **первый символ текста**.

При прочтении файла до конца с помощью вызова методов `read()`, `readlines()` курсор перемещается в **конец файла** и последующие чтения ничего не дают.

При использовании метода `readline()` текущая позиция перемещается на **следующую строку** файла.

После завершения чтения мы больше не можем считать ни одного символа из файла. Все последующие вызовы методов `read()` или `readline()` будут приводить к считыванию **пустой строки**.

Для повторного чтения данных из файла, можно:

- переоткрыть файл, тогда курсор снова попадёт в начало;
- переместить курсор с помощью файлового метода `seek()`.

**Файловый метод file.seek(n)** — задаёт позицию курсора в байтах от начала файла. Чтобы перевести курсор в самое начало файла необходимо передать методу значение `0`.

```py
with open('input.txt') as file:
    data = file.readlines()
    file.seek(0) # перемещаем курсор в начало файла
```

Необходимо быть осторожным с символами, использующими более `1` байта (кириллица в кодировке `utf-8`), обращение к "промежуточному" байту может вызвать **ошибку**.

---

**`Файловый метод filetell()`** — получает позицию курсора в файле.

```py
with open('input.txt') as file:
    file.tell() # 0
```

# Декораторы

**Декоратор** — это функция, которая принимает другую функцию, расширяет её возможности и возвращает результат, не меняя исходный код самой функции.

**В основе декораторов лежат два основных понятия:**

- функции являются объектами первого класса (их можно передавать как аргументы);
- концепция замыкания.

Ознакомиться с этими понятиями вы можете [здесь](functions.md#функции-высшего-порядка)

**Базовая структура декоратора:**

```py
def my_decorator(func):
    def wrapper():
        print("Действие ДО вызова функции")
        func()
        print("Действие ПОСЛЕ вызова функции")
    return wrapper

@my_decoratordef say_hello():
    print("Привет!")

say_hello()
```

Синтаксический сахар `@name` над определением функции эквивалентен записи `say_hello = my_decorator(say_hello)`. Это позволяет «оборачивать» функции в новую логику максимально наглядно.

Если декорируемая функция принимает аргументы, в обертке `wrapper` необходимо использовать `*args` и `**kwargs`. Это делает декоратор универсальным для любых функций.

```py
def logger(func):
    def wrapper(*args, **kwargs):
        print(f"Вызов {func.__name__} с аргументами: {args}, {kwargs}")

        return func(*args, **kwargs)

    return wrapper
```

При использовании декораторов возникает нюанс: метаданные исходной функции (её имя **name** и документация **doc**) заменяются данными обертки `wrapper`. Чтобы этого избежать, внутри декоратора используют стандартный декоратор `@functools.wraps(func)`, который копирует метаданные оригинала.

```py
import functools

def logger(func):

    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        print(f"Вызов {func.__name__} с аргументами: {args}, {kwargs}")

        return func(*args, **kwargs)

    return wrapper
```

У одной функции может и несколько декораторов. Если над функцией указано несколько `@name`, они применяются снизу вверх: ближний к функции оборачивает её первым, а затем его результат оборачивается следующим.

```py
def bold(func):
    def wrapper():
        print("<b>", end="")      # Начало внешнего слоя
        res = func()
        print("</b>")     # Конец внешнего слоя
        return res
    return wrapper

def italic(func):
    def wrapper():
        print("<i>", end="")      # Начало внутреннего слоя
        res = func()
        print("</i>", end="")     # Конец внутреннего слоя
        return res
    return wrapper

@bold
@italic
def hello():
    print("Hello, World!", end="")

hello() # <b><i>Hello, World!</i></b>
```

**Особенности работы с декораторами:**

- Декораторы замедляют вызов функции;

- Раздекорировать функцию невозможно. Безусловно, существуют способы, позволяющие создать декоратор, который можно отсоединить от функции, но это считается плохой практикой.

- Декораторы оборачивают функции, что может затруднить отладку(частично решается добавлением `@functools.wraps`).

# Дата и время в Python

## Модуль datetime

В Python работа с датой и временем осуществляется в основном через встроенный модуль datetime, который предоставляет классы и функции для:

- **Представления моментов времени** — конкретные даты и время;
- **Работы с интервалами** — промежутки времени между событиями;
- **Форматирования и парсинга** — преобразование между строками и объектами времени;
- **Арифметических операций** — сложение, вычитание временных периодов.

## Основные классы модуля datetime

Модуль datetime — основной инструмент для работы с датой и временем в Python. Он содержит несколько важных классов:

**`datetime.date()`** — объект, который хранит только дату: год, месяц и день. Он инкапсулирует в себе три атрибута: year, month и day.

```py
from datetime import date

d = date(2026, 2, 12) # 2026-02-12
```

- `d.year` ­— год;
- `d.month` — месяц;
- `d.day` — день.

---

**`d.today()`** — возвращает текущую дату.

```py
from datetime import date

today = date.today() # 2026-02-12 

```

---

**`datetime.time()`** — объект представляет время суток, изолированное от какой-либо конкретной даты. Он хранит hour, minute, second, microsecond.

```py
from datetime import time

# Создаем объект, представляющий время 16:30
t = time(16, 30, 0)
```

- `t.hour` — часы;
- `t.minute` — минуты;
- `t.second` — секунды;
- `t.microsecond` — микросекунды.

---

**`datetime.datetime()`** — объединяет в себе информацию как из date, так и из time. Он хранит все атрибуты: от года до микросекунд.

```py
from datetime import datetime

dt_manual = datetime(2026, 2, 12, 18, 53, 0) # 2026-02-12 18:53:00
```

- `dt.year` ­— год;
- `dt.month` — месяц;
- `dt.day` — день.
- `dt.hour` — часы;
- `dt.minute` — минуты;
- `dt.second` — секунды;
- `dt.microsecond` — микросекунды.

---

**`datetime.datetime.now()`** — метод, позволяющий получить текущее локальное время (возвращает объект datetime).

```py
from datetime import datetime

dt_now = datetime.now() # текущее дата и время
```

---

**`datetime.datetime.now(datetime.UTC)`** — возвращает текущее время в UTC (naive).

```py
from datetime import datetime

utc_time = datetime.datetime.now(datetime.UTC)

print(utc_time) # 2026-02-12 16:42:52.253140
```

## Сложение и вычитание дат (timedelta)

**Объекты datetime поддерживают следующие операции**:

```py
from datetime import datetime

d1 = datetime(2026, 2, 12)
d2 = datetime(2026, 2, 13)

print(d2 - d1) # вычитание: 1 day, 0:00:00
print(d2 > d1) # сравнение: True
print(d2 < d1) # сравнение: False
```

Результатом вычитания двух объектов datetime является объект класса **timedelta**, который хранит разницу в днях и секундах.

- delta.days — содержит количество полных суток в интервале;
- delta.seconds — оставшееся количество секунд (от 0 до 86399).

**Объект timedelta** представляет собой не конкретный момент времени, а длительность или разницу между двумя датами. Эта длительность может быть выражена в днях, секундах и микросекундах.

**К объекту datetime можно прибавлять timedelta и вычитать timedelta из него**:

```py
from datetime import datetime, timedelta

d1 = datetime(2026, 2, 12)
delta = timedelta(days=15, hours=3)

print(d1 + delta) # 2026-02-27 03:00:00
print(d1 - delta) # 2026-01-27 21:00:00
```

**Метод `delta.total_seconds()`** — возвращает полное количество секунд timedelta.

## Методы strftime и strptime

**Метод strftime** (string format time) — вызывается у объекта datetime и принимает на вход строку с кодами форматирования. Эти коды определяют, как будет выглядеть итоговая текстовая строка.

Наиболее часто используемые коды:

- `%Y` — год (четырехзначное);
- `%m` — номер месяца с ведущим нулем (от 01 до 12);
- `%d` — день месяца с ведущим нулем (от 01 до 31);
- `%H` — час в 24-часовом формате (от 00 до 23);
- `%M` — минута (от 00 до 59);
- `%S` — секунда (от 00 до 61).

```py
from datetime import datetime

d1 = datetime(2026, 2, 12, 18, 53, 13)

formatted_string_iso = d1.strftime("%Y-%m-%dT%H:%M:%S")

print(formatted_string_iso) # 2026-02-12T18:53:13
```

---

**Метод strptime** (string parse time) — принимает два аргумента: исходную строку с датой и строку с кодами форматирования, которая в точности описывает структуру исходной строки.

```py
from datetime import datetime

date_string_from_api = "2026-02-12 15:00:00"

dt = datetime.strptime(date_string_from_api, "%Y-%m-%d %H:%M:%S")

print(dt) # 2026-02-12 15:00:00
```

## Абсолютное, относительное время. Модуль zoneinfo

**Абсолютные даты (naive datetime)** — не содержат информацию, которая бы могла определить часовой пояс или летнее время.

**Относительные даты (aware datetime)** — содержат достаточно информации для определения часового пояса или отслеживания изменений из-за летнего времени.

Для работы с относительными датами в стандартной библиотеке Python есть модуль **zoneinfo**.

```py
from datetime import datetime
from zoneinfo import ZoneInfo

dt = datetime.now(ZoneInfo("Europe/Riga"))
print(dt) # 2026-02-12 18:33:03.518374+02:00
```

Внутри объекта dt хранится информация:

- дата и время
- таймзона
- информация о переходах на летнее/зимнее время

# Формат JSON

## Что такое JSON?

**JSON (JavaScript Object Notation)** – это общий формат для представления значений и объектов, используемый для передачи данных.

**Особенности:**

- **Простота и читаемость** — синтаксис JSON минималистичен и интуитивно понятен.
- **Универсальность** — большинство современных языков, включая Python, Java, C# и другие, имеют встроенные или сторонние библиотеки для удобной работы с этим форматом.
- **Широкое применение** — используется повсеместно: от передачи данных между браузером и сервером (API) до хранения настроек в конфигурационных файлах и работы с базами данных.

**Сериализация** — это процесс преобразования структуры данных Python (например, словаря или списка) в текстовую строку формата JSON.

**Десериализация** — это обратный процесс: преобразование JSON-строки в объект Python.

## Модуль json в Python

**Модуль json** — модуль стандартной библиотеки Python, позволяющий работать с данными в формате JSON.

**Функция `json.dumps(obj)`** - берет объект Python и возвращает строку в формате JSON.

**Функция `json.loads(str)`** — парсит строку, содержащую JSON, и возвращает объект Python (чаще всего словарь или список).

**Функция `json.dump(obj, file)`** — сериализует объект Python и записывает его напрямую в файловый объект. Она принимает два обязательных аргумента: сам объект и файловый объект, открытый на запись.

**Функция `json.load(file)`** — читает данные из файлового объекта, содержащего JSON, и десериализует и возвращает их в объект Python.

## Продвинутая работа с JSON

**Работа с пользовательскими объектами Python:**

Имеется класс User, необходимо сериализовать его.

```py
import json

class User:
    def __init__(self, name, email):
        self.name = name
        self.email = email

user = User("Алиса", "alice@example.com")

# Попытка сериализации
# json.dumps(user)
# -> TypeError: Object of type User is not JSON serializable
```

Это можно сделать несколькими способами:

Параметр `default` функции `json.dumps()` — позволяет вызывать функцию переданную в него для обработки всех объектов, которые модуль не может сериализовать самостоятельно. Функция должна возвращать JSON-совместимое представление объекта, например, словарь.

```py
def user_serializer(obj):
    if isinstance(obj, User):
        return {"name": obj.name, "email": obj.email}
    # Для других типов данных, которые могут вызвать ошибку,
    # вызываем стандартный обработчик
    raise TypeError(f"Object of type {type(obj).__name__} is not JSON serializable")

json_string = json.dumps(user, default=user_serializer, indent=4, ensure_ascii=False)
print(json_string)
```

---

Более лаконичным вариантом может быть использование `__dict__`, который вернет все атрибуты объекта в виде словаря.

```py
json.dumps(user, default=lambda o: o.__dict__)
```

---

**Cобственный JSONEncoder** — собственный класс-кодировщик, унаследованный от `json.JSONEncoder` с переопределенным методом `default()`, для более сложных и системных задач.

```py
from json import JSONEncoder

class CustomEncoder(JSONEncoder):
    def default(self, obj):
        if isinstance(obj, User):
            return {"name": obj.name, "email": obj.email}
        # Вызываем родительский метод для обработки стандартных типов
        # или выброса TypeError
        return super().default(obj)

json_string = json.dumps(user, cls=CustomEncoder, indent=4, ensure_ascii=False)
print(json_string)
```

---

Как из JSON-строки получить не просто словарь, а экземпляр класса User? Для этого в функциях `json.loads()` и `json.load()` существует параметр `object_hook`.

**`object_hook`** — это функция, которая будет вызвана для каждого декодированного JSON-объекта. Она получает словарь в качестве аргумента, и возвращает нужный нам объект Python.

```py
def user_decoder(dct):
    # Проверяем, похож ли словарь на нашего пользователя (например, по ключам)
    if "name" in dct and "email" in dct:
        return User(dct["name"], dct["email"])
    return dct

json_string = '{"name": "Алиса", "email": "alice@example.com"}'

# Десериализуем строку с использованием object_hook
user_obj = json.loads(json_string, object_hook=user_decoder)
```

---

Обработка ошибок JSON с помощью конструкции try...except.

```py
import json

invalid_json = '{"name": "Боб", "age": 40,}' # Лишняя запятая в конце

try:
    data = json.loads(invalid_json)
    print("JSON успешно разобран:", data)
except json.JSONDecodeError as e:
    print("Ошибка декодирования JSON!")
    print(f"Сообщение об ошибке: {e.msg}")
    print(f"Строка: {e.lineno}, колонка: {e.colno}")
```

## Производительность и альтернативы

Для большинства повседневных задач производительности встроенного модуля json более чем достаточно.

Однако в высоконагруженных системах, при обработке гигантских JSON-файлов или в задачах, критичных ко времени отклика (например, в веб-фреймворках), он может стать узким местом.

**Альтернативных библиотеки:**

**orjson**:

- высокая скорость
- нативная поддержка типов (dataclasses, datetime и т.д)
- надежность

**msgspec**:

- экстремальная производительность
- валидация
- эффективность использования памяти
- несколько форматов

# Пакетный менеджер pip и виртуальное окружение venv

**Пакетный менеджер pip** — это стандартная система управления пакетами. Она позволяет скачивать, устанавливать и обновлять библиотеки из официального репозитория PyPI.

**Основные команды pip:**

- `pip install <package>` — установка библиотеки;
- `pip uninstall <package>` — удаление;
- `pip list` — список всех установленных пакетов в текущем окружении;
- `pip freeze > requirements.txt` — сохранение списка всех зависимостей проекта в файл. Это стандарт индустрии: другой разработчик сможет установить их одной командой `pip install -r requirements.txt`.

`venv` — это модуль для создания виртуальных окружений. Каждое окружение — это изолированная папка с собственной копией интерпретатора Python и своими библиотеками.

**Зачем это нужно:**

- **Конфликты версий** : одному проекту нужна библиотека версии 1.0, а другому — 2.0. Виртуальные окружения позволяют им не пересекаться;
- **Чистота системы** : вы не захламляете глобальный Python десятками библиотек, которые нужны только для одного эксперимента.

**Жизненный цикл окружения:**

1. Создание:

   - `python -m venv .venv` (где `.venv` — название папки, обычно её скрывают точкой).

2. Активация:

    - Windows: `.venv\Scripts\activate`

    - macOS/Linux: `source .venv/bin/activate`

      После этого в терминале появится префикс (`.venv`), означающий, что теперь все команды pip работают только внутри этой папки.

3. Деактивация:

   - Команда `deactivate` возвращает вас в глобальную среду.

>Важно помнить: папку с виртуальным окружением (`.venv`) никогда не добавляют в [Git](glossary.md#git) (её прописывают в файле `.gitignore`). Вместо неё передают файл `requirements.txt`, по которому окружение воссоздается за секунды.

# Парадигмы программирования

## Что такое парадигмы программирования

**Парадигма программирования** — совокупность идей и понятий, определяющих стиль написания компьютерных программ.

**Парадигма программирования определяется:**

- вычислительной моделью;
- базовой программной единицей (-ами);
- методами разделения абстракций.

Язык программирования не обязательно использует единственную парадигму. Существуют [мультипарадигменные языки](glossary.md#мультипарадигменный-язык).

Python — мультипарадигменный язык программирования.

## Основные парадигмы программирования

**Императивное программирование** — парадигма программирования, в которой программа описывается как **последовательность команд, изменяющих состояние программы**.

**Императивное программирование характеризуется** тем, что:

- в исходном коде программы записаны **инструкции** (команды);
- инструкции должны выполняться **последовательно**;
- данные, полученные при выполнении инструкции, могут **записываться в память**;
- данные, получаемые при выполнении предыдущих инструкций, могут **читаться из памяти** последующими инструкциями.

При императивном подходе к составлению кода широко используется **присваивание**. Наличие **операторов присваивания** увеличивает сложность модели вычислений и создает условия для специфических ошибок императивных программ.

**Основные механизмы управления:**

- последовательное исполнение команд;
- использование именованных переменных;
- использование оператора присваивания;
- использование ветвления (оператор if);
- использование безусловного перехода (goto).

Ключевой идеей императивного программирования является работа с переменными, как с временным хранением данных в оперативной памяти.

---

**Структурное программирование** — парадигма программирования, основанная на использовании трёх базовых **управляющих конструкций** и **отказе от оператора безусловного перехода (goto)**.

Парадигму структурного программирования предложил нидерландский ученый **Эдсгер Дейкстра**.

**Любой алгоритм можно выразить с помощью трёх структур:**

- Последовательность  
- Ветвление  
- Цикл

**Основные механизмы управления:**

- **последовательное исполнение** команд;
- использование **именованных переменных**;
- использование **оператора присваивания**;
- использование **ветвления** (оператор if);
- использование **циклов**;
- использование **подпрограмм** (функций).

В структурном программировании программа по возможности разбивается на маленькие подпрограммы (функции) с изолированным контекстом.

---

**Объектно-ориентированное программирование** — парадигма программирования, в которой программа строится вокруг **объектов** —  струк­ту­р дан­ных, со­стоя­щих из **по­лей**, опи­сы­ваю­щих со­стоя­ние, и **ме­то­дов** – функций, при­ме­няе­мых к объ­ек­там для изме­не­ния или за­про­са их со­стоя­ния.

**Четыре столпа ООП:**

- **Абстракция** — скрытие сложности, предоставление простого интерфейса.
- **Полиморфизм** — один интерфейс — разные реализации.
- **Инкапсуляция** — объединение данных и методов, сокрытие внутреннего устройства.
- **Наследование** — возможность создавать новый класс на основе существующего.

Объектно-ориентированное программирование — это способ организовать код так, чтобы он отражал **структуру предметной области через объекты**, которые хранят данные и умеют что-то делать.

Подробнее об этом [здесь](oop.md).

---

**Логическое программирование** — это парадигма программирования, в которой программа описывается как **набор логических утверждений** (фактов и правил), а вычисления происходят путём **логического вывода**.

В конце 1960-х годов **Корделл Грин** предложил использовать резолюцию как основу логического программирования. **Алан Колмеро** создал язык логического программирования `Prolog` в 1971 году.

**Ключевые особенности:**

- **Декларативность** — описывается что есть, а не как вычислять.
- **Обратимость** — многие предикаты работают «в обе стороны»: можно искать вход по выходу и наоборот.
- **Недетерминизм** — система может находить все возможные решения.
- **Унификация** — сопоставление с образцом на основе логики (не просто присваивание).

Важное его преимущество — достаточно высокий уровень машинной независимости, а также возможность откатов, возвращения к предыдущей подцели при отрицательном результате анализа одного из вариантов в процессе поиска решения.

Один из концептуальных недостатков логического подхода — специфичность класса решаемых задач.

---

**Функциональное программирование** — парадигма программирования, в которой **вычисления трактуются как вычисление математических функций**, а изменение состояния и побочные эффекты **исключаются или сводятся к минимуму**.

**Основная идея:**  

- **Программа** — это набор чистых функций, которые преобразуют данные, не изменяя их.
- **Неизменяемые переменные** — в функциональном программировании можно определить переменную, но изменить ее значение нельзя;
- **Чистая функция** — это функция, результат работы которой предсказуем. При вызове с одними и теми же аргументами, такая функция всегда вернет одно и то же значение. Про такие функции говорят, что они не вызывают побочных эффектов;
- **Функции высшего порядка** — могут принимать другие функции в качестве аргумента или возвращать их;
- **Рекурсия** — поддерживается многими языками программирования, а для функционального программирования обязательна. В языках ФП отсутствуют циклы, поэтому для повторения операций служит рекурсия.
- **Лямбда-выражения** — способ определения анонимных функциональных объектов.

# Алгоритмы сортировки

## Общие сведения

**Алгоритм сортировки** — это алгоритм упорядочивания элементов в структурах данных. Алгоритмы сортировки оцениваются по скорости выполнения и эффективности использования памяти.

**Сортировка на месте** ­— алгоритмы сортировки, не потребляющие дополнительной памяти.

**Основные алгоритмы сортировки:**

**Медленные:**

- Пузырьковая сортировка (Bubble sort);
- Сортировка выбором (Selection sort);
- Сортировка простыми вставками (Insertion sort).

**Быстрые:**

- Сортировка Шелла (Shell sort);
- Быстрая сортировка (Quick sort);
- Сортировка слиянием (Merge sort);
- Пирамидальная сортировка (Heap sort);
- Сортировка TimSort (используется в Java и Python).

**Адаптивные алгоритмы сортировки:**

- Сортировка подсчетом (Counting sort)
- Блочная сортировка (Bucket sort)
- Поразрядная сортировка (Radix sort)

Адаптивные алгоритмы учитывают особенности входных данных и работают быстрее в определенных случаях.

---

## Базовые алгоритмы

**Сортировка пузырьком:**

```py
l = [8, 4, 7, 3, 1]

d = len(l)

for i in range(d - 1):
    for j in range(d - 1 - i):
        if l[j] > l[j + 1]:
            l[j], l[j + 1] = l[j + 1], l[j]
        print(l)
```

В примере выше мы сортируем список `l` методом пузырька. Разберем основные этапы сортировки:

1. `d = len(l)` — для удобства мы создадим переменную `d` и присвоим ей значение длины нашего списка `len(l)`. Длина списка `l = [8, 4, 7, 3, 1]` равна 5.

2. `for i in range(d - 1):` — создадим цикл `for` с переменной-счетчиком `i`, в диапазоне функции `range(d - 1)`. То есть переменная `i` будет принимать значения от `0` до `4` невключительно(`0, 1, 2, 3`).

3. `for j in range(d - 1 - i):` — создадим вложенный цикл `for` с переменной-счетчиком `j`, в диапазоне функции `range(d - 1 - i)`. Значением `i` в первой итерации цикла равняется `0`, а значит `j` работает в диапазоне от `0` до `4` невключительно(`0, 1, 2, 3`).

4. `if l[j] > l[j + 1]:` — с помощью условной конструкции `if` проверяем значение списка `l` по индексу `j`(то есть `l[0] = 8`). Если значение `l[j]` больше, чем `l[j + 1]`(`l[1] = 4`), то условие истинно и мы переходим к выполнению тела условия.

5. `l[j], l[j + 1] = l[j + 1], l[j]` — на этом этапе мы с помощью [множественного присваивания](operators.md#операторы-присваивания) меняем местами значения в списке. То есть `l[j] = 8` теперь равно `j[j + 1] = 4`, а `l[j + 1] = 4` теперь равно `l[j] = 8`. Исходный список теперь выглядит так: `[4, 8, 7, 3, 1]`.

6. Далее мы переходим к следующей итерации вложенного цикла и выполняем этапы 3, 4 и 5 снова. С каждой итерацией сравниваются пары значений списка и большее значение двигается в правую сторону. То есть на выходе из вложенного цилка список будет выглядеть так: `[4, 7, 3, 1, 8]`. Как видно элемент `8` занял свою позицию в списке. Внешний цикл с переменной `i` переходит к своей следующей итерации.

7. Когда внешний цикл переходит к своей следующей итерации, увеличивается значение переменной `i`. И соотвественно диапазон `d - 1 - i` вложенного цикла с переменной `j` уменьшается. Таким образом мы откидываем отсортированную часть списка и наш цикл работает только с неотсортированными значениями. То есть `[4, 7, 3, 1]`, элемент `[8]` принадлежит отсортированной части списка, и в сортировке больше не принимает участия.

8. Таким образом сортировка проходит по каждому этапу. Для наглядности, вот как выглядит исходный список на каждом этапе сравнения пар элементов.

    ```py
    [8, 4, 7, 3, 1] # исходный список
    # Первая итерация внешнего цикла (i = 0)
    [4, 8, 7, 3, 1] # 8 > 4
    [4, 7, 8, 3, 1] # 8 > 7
    [4, 7, 3, 8, 1] # 8 > 3
    [4, 7, 3, 1, 8] # 8 > 1
    # Вторая итерация внешнего цикла (i = 1)
    [4, 7, 3, 1, 8] # 4 < 7
    [4, 3, 7, 1, 8] # 7 > 3
    [4, 3, 1, 7, 8] # 7 > 1
    # Третья итерация внешнего цикла (i = 2)
    [3, 4, 1, 7, 8] # 3 < 4 
    [3, 1, 4, 7, 8] # 4 > 1
    # Четвертая итерация внешнего цикла (i = 3)
    [1, 3, 4, 7, 8] # 1 < 3
    ```

Сортировку пузырьком можно оптимизировать. Как мы видели в примере выше каждую итерацию внешнего цикла значения в списке менялись местами. Но что если представить, что наш список `l` изначально выглядел таким образом: `[8, 1, 3, 4, 7]`. Как вы можете заметить в этом списке есть подсписок уже осортированных значений `[1, 3, 4, 7]`. То есть большая часть списка уже отсортированы. Алгоритму нужно только разместить элемент `8` в конец списка и список будет полностью отсортирован. Но текущий алгоритм после размещения элемента `8` в конец списка будет дальше сравнивать значения в списке, даже если он уже полностью отсортирован.

Попробуем посмотреть каждый этап сортировки такого списка текущим алгоритмом.

```py
[8, 1, 3, 4, 7] # исходный список
# Первая итерация внешнего цикла (i = 0)
[1, 8, 3, 4, 7] # 8 > 1
[1, 3, 8, 4, 7] # 8 > 3
[1, 3, 4, 8, 7] # 8 > 4
[1, 3, 4, 7, 8] # 8 > 7
# Вторая итерация внешнего цикла (i = 1)
[1, 3, 4, 7, 8] # 1 < 3
[1, 3, 4, 7, 8] # 3 < 4
[1, 3, 4, 7, 8] # 4 < 7
# Третья итерация внешнего цикла (i = 2)
[1, 3, 4, 7, 8] # 1 < 3
[1, 3, 4, 7, 8] # 3 < 4
# Четвертая итерация внешнего цикла (i = 3)
[1, 3, 4, 7, 8] # 1 < 3
```

Как вы можете видеть внешний цикл совершает еще три итерации и сравнивает значения, хотя список уже полностью отсортирован. В таком случае будет логичным доработать алгоритм, чтобы после полной отсортировки алгоритм останавливался и выдавал результат.

Сделать это можно с помощью перменной флага `isDone`:

```py
l = [8, 1, 3, 4, 7]

d = len(l)

for i in range(d - 1):
    isDone = True
    for j in range(d - 1 - i):
        if l[j] > l[j + 1]:
            l[j], l[j + 1] = l[j + 1], l[j]
            isDone = False
    if isDone:
        break
print(l)
```

В примере выше **при каждой новой итерации внешнего цикла переменной `isDone` присваивается значение `True`**. Если во время выполнения итерации, во вложенном цикле выполнится условие, то переменная **примет значение `False`**. То есть, если итерация внешнего цикла заканчивается без каких либо изменений в списке(то есть список уже полностью отсортирован), то значение переменной останется `True`, и когда она будет проходить конструкцию `if isDone:`, условие будет истинным и цикл остановится командой `break`. Таким образом мы можем избежать выполнения лишних итераций в цикле и пустых проверок.

```py
[8, 1, 3, 4, 7] # исходный список
# Первая итерация внешнего цикла (i = 0)
[1, 8, 3, 4, 7] # 8 > 1
[1, 3, 8, 4, 7] # 8 > 3
[1, 3, 4, 8, 7] # 8 > 4
[1, 3, 4, 7, 8] # 8 > 7
# Цикл продложает работу, так как в списке были изменения и переменная isDone = False
# Вторая итерация внешнего цикла (i = 1)
[1, 3, 4, 7, 8] # 1 < 3
[1, 3, 4, 7, 8] # 3 < 4
[1, 3, 4, 7, 8] # 4 < 7
# Цикл прерывается, так как изменений в списке не последовало и значением переменной isDone осталось True.
```

**Примечания:**

- Алгоритм сортировки пузырьком считается учебным и практически не применяется вне учебной литературы, а на практике применяются более эффективные.

---

**Сортировка выбором:**

```py
l = [8, 4, 7, 3, 1]

d = len(l)

for i in range(d - 1):
    mx = d - 1 - i
    for j in range(d - 1 - i):
        if l[j] > l[mx]:
            mx = j
    l[d - 1 - i], l[mx] = l[mx], l[d - 1 - i]

print(l)
```

В примере выше мы сортируем список `l` сортировкой выбором. Разберем основные этапы сортировки:

1. `d = len(l)` — для удобства мы создадим переменную `d` и присвоим ей значение длины нашего списка `len(l)`. Длина списка `l = [8, 4, 7, 3, 1]` равна 5.

2. `for i in range(d - 1):` — создадим цикл `for` с переменной-счетчиком `i`, в диапазоне функции `range(d - 1)`. То есть переменная `i` будет принимать значения от `0` до `4` невключительно(`0, 1, 2, 3`).

3. В теле внешнего цикла каждую итерацию мы будем присваивать переменной `mx` значение `d - 1 - i`. В первой итерации это значение `5 - 1 - 0` равно `4`, так как `i = 0`. Таким образом каждую итерацию переменная `mx` будет содержать индекс последнего элемента неотсортированной части списка(в нашем случае этот элемент `l[4] = 1`).

4. `for j in range(d - 1 - i):` — создадим вложенный цикл `for` с переменной-счетчиком `j`, в диапазоне функции `range(d - 1 - i)`. Значением `i` в первой итерации цикла равняется `0`, а значит `j` работает в диапазоне от `0` до `4` невключительно(`0, 1, 2, 3`). Обратим внимание, что последний элемент неотсортированной части списка не попадает в этот диапазон. Он уже присвоен переменной `mx`, а сравнивать его с самим собой это одна лишняя итерация вложенного цикла.

5. `if l[j] > l[mx]:` — с помощью условной конструкции `if` проверяем значение списка `l` по индексу `j`(`l[0] = 8`). Если значение `l[j]` больше, чем `l[mx]`(`l[4] = 1`), то условие истинно и мы переходим к выполнению тела условия.

6. `mx = j` — в теле условия мы обновляем значение переменной `mx`, поскольку `mx` должна хранить индекс максимального элемента списка.

7. Далее мы переходим к следующей итерации вложенного цикла и выполняем этапы 4, 5 и 6 снова. С каждой итерацией сравнивается значение `l[mx]` и `l[j]`. В случае, если мы находим значение больше чем `l[mx]` мы обновляем переменную `mx`, записывая в нее новый индекс максимального элемента в списке. В нашем списке `[8, 4, 7, 3, 1]` нет элемента больше, чем `8`, поэтому переменная `mx` по окончанию вложенного цикла будет иметь значение `0`.

8. `l[d - 1 - i], l[mx] = l[mx], l[d - 1 - i]` — после выхода из вложенного цикла с помощью множественного присваивания, мы присваиваем последнему элементу неотсортированного списка — `l[d - 1 - i]`, значение максимального элемента списка — `l[mx]`, а `l[mx]` получает значение `l[d - 1 - i]`. То есть в нашем случае `l[4] = l[0]`, а `l[0] = l[4]`. Таким образом в конце первой итерации внешнего цикла исходных список будет иметь вид: `[1, 4, 7, 3, 8]`. Алгоритм нашел максимальный элемент `8` и разместил его в конце списка.

9. Когда внешний цикл переходит к своей следующей итерации, увеличивается значение переменной `i`. И соотвественно диапазон `d - 1 - i` вложенного цикла с переменной `j` уменьшается. Таким образом мы откидываем отсортированную часть списка и наш цикл работает только с неотсортированными значениями. То есть `[1, 4, 7, 3]`, элемент `[8]` принадлежит отсортированной части списка, и в сортировке больше не принимает участия.

10. Таким образом сортировка проходит по каждому этапу. Цель каждой итерации внешнего цикла: найти максимальный элемент неотсортированной части списка и разместить его в конец неотсортированной части. Для наглядности вот как выглядит исходный список в конце каждой итерации внешнего цикла.

    ```py
    [8, 4, 7, 3, 1] # исходный список
    # Первая итерация внешнего цикла (i = 0)
    [1, 4, 7, 3, 8] # l[mx] = 8
    # Вторая итерация внешнего цикла (i = 1)
    [1, 4, 3, 7, 8] # l[mx] = 7
    # Третья итерация внешнего цикла (i = 2)
    [1, 3, 4, 7, 8] # l[mx] = 4
    # Четвертая итерация внешнего цикла (i = 3)
    [1, 3, 4, 7, 8] # l[mx] = 3
    ```

**Примечания:**

- Алгоритм сортировки выбором также считается учебным и практически не применяется вне учебной литературы. На практике используют более эффективные алгоритмы.

---

**Сортировка простыми вставками:**

```py
l = [8, 10, 7, 3, 1]

d = len(l)

for i in range(1, d): 
    elem = l[i]
    j = i

    while j >= 1 and l[j - 1] > elem:
        l[j] = l[j - 1]
        j -= 1
    l[j] = elem

print(l)
```

В примере выше мы сортируем список `l` сортировкой простыми вставками. Разберем основные этапы сортировки:

1. `d = len(l)` — для удобства мы создадим переменную `d` и присвоим ей значение длины нашего списка `len(l)`. Длина списка `l = [8, 10, 7, 3, 1]` равна 5.

2. `for i in range(d - 1):` — создадим цикл `for` с переменной-счетчиком `i`, в диапазоне функции `range(1, d)`. То есть переменная `i` будет принимать значения от `1` до `5` невключительно(`1, 2, 3, 4`). Таким образом мы разделяем наш список на две части. Первая часть — `l[0:1] = [8]`, и вторая часть — `l[1:] = [10, 7, 3, 1]`.

3. `elem = l[i]` — созадем переменную `elem` и присваиваем ей значение `l[i]`(`l[1] = 10`). Переменная `elem`, каждую итерацию внешнего цикла, хранит первый элемент неотсортированной части списка.

4. `j = i` — создаем переменную `j` — счетчик вложенного цикла и присваиваем ей значение переменной `i`(`j = 1`).

5. `while j >= 1 and l[j - 1] > elem:` — создаем цикл `while` с условием: пока переменная `j` больше либо равна `1`, и значение списка `l` по индексу `j - 1` больше, чем переменная `elem`, выполняется тело цикла. В нашей первой итерации цикла значение переменной `elem`(`10`) больше, чем значение `8` которое уже находится у нас в отсортированной части списка, а значит условие ложно и мы пропускаем тело цикла и вернемся к нему во второй итерации внешнего цикла `for`.

6. `l[j] = elem` — как мы помним переменная `elem` хранит значение элемента списка по индексу `l[i]`(`10`). То есть в конце первой итерации мы присваиваем `l[j]` значение `l[i]`. Что есть `l[1]` = `l[1]`. То есть элемент не меняет своей позиции и список остается неизменным(`[8, 10, 7, 3, 1]`).

7. Переходим ко второй итерации внешнего цикла. Теперь значение переменной `i = 2`, а значит `elem = l[2]`(`7`). Переменная `j = i` и равна `2`. Наш список снова делится на 2 части. Первая часть `[8, 10]` — **отсортированная**(все элементы стоят по возрастанию), вторая часть `[7, 3, 1]` — **неотсортированная**. Теперь перейдем к вложенному циклу `while`. Переменная `j = 2` и проходит первое условие цикла (`j >= 1`). Значение списка `l` по индексу `j - 1`, то есть `10`(`l[1]`), больше, чем значение `elem` равное `7`, а значит оба условия верны и выполняется тело цикла.

8. `l[j] = l[j - 1]` — на этом этапе мы присваиваем `l[j]`(`l[2]`) значение под индексом `l[j - 1]`(`l[1]`). То есть в нашем случае `l[2] = 10` и список преобретает вид `[8, 10, 10, 3, 1]`. Обратите внимание, что мы клонировали элемент `10` на один индекс вправо, но при этом не потеряли значение `7`, ведь оно хранится у нас в переменной `elem`.

9. `j -= 1` — отнимаем 1 от переменной `j`, чтобы перейти к следующему элементу в нашей отсортированной части списка `[8, 10]`. В нашем случае `j - 1 = 1`. `l[j - 1]`(`l[0] = 8`), больше чем `elem = 7`, поэтому мы переходим к следующей итерации цикла `while`. В этой итерации мы клонируем уже элемент `l[0] = 8` вправо на один индекс и список преобретает вид [`8, 8, 10, 3, 1`]. Снова отнимаем от переменной `j -= 1`  и переходим к следующей итерации цикла.

10. Переменная `j` не проходит условие цикла так как `j - 1 = 0`, а по условию цилка `j >= 1` значение `j` должно быть больше, либо равно `1`.  Цикл завершается и мы переходим к концу второй итерации внешнего цикла `for` и присваиваем `l[j]`(`l[0]`) значение переменной `elem`(`7`). То есть на выходе из второй итерации внешнего цикла список преобретает вид `[7, 8, 10, 3, 1]`.

11. Далее мы продолжаем двигаться по списку и перемещать элементы из неотсортированной части в отсортированную.

12. Как вы могли заметить, смысл алгоритма в том, чтобы взять переменную из неотсортированной части списка и найти ей место в уже отсортированной части. Поскольку мы знаем, что все элементы отсортированной части уже стоят в правильном порядке, то просто двигаем `elem` пока не найдется элемент меньший чем он. После того как мы нашли этот элемент, нам нет необходимости сравнивать `elem`, со всеми остальными элементами отсортированной части, потому что мы уже знаем, что элементы далее будут меньше.

---

Алгоритм сортировки простыми вставками делает меньше сравнений, чем алгоритм сортировки пузырьком или выбором. Более того мы можем оптимизировать этот алгоритм, чтобы сократить количество сравнений еще больше. Сделать это можно с помощью **бинарного поиска**.

**Бинарный поиск** — это **эффективный алгоритм поиска позиции заданного элемента** (или места для его вставки) **в отсортированной** последовательности, основанный на последовательном **делении области поиска пополам**. На каждом шаге алгоритм сравнивает искомое значение со с**редним элементом текущего диапазона и отбрасывает половину элементов**, в которой искомый элемент **заведомо отсутствовать не может**.  

```py
left, right = 0, i

while left < right:
    mid = (left + right) // 2
    if l[mid] > elem:
        right = mid
    else:
        left = mid + 1
```

В примере выше переменна `i` — это индекс элемента в массиве от которого отталкивается поиск. Переменные `left` и `right` это границы диапазона поиска нужного индекса. Цикл продолжается пока граница `left` меньше границы `right`. При этом каждую итерацию алгоритм находит среднее значение `mid` в диапазоне `[left:right]` и если `l[mid]` больше, чем `elem`, то граница диапазона `right` смещается до `mid`. Если же `l[mid]` меньше, либо равен `elem`, левая граница смещается к `mid + 1`. Таким образом, по окончанию алгоритма в переменной `left` хранится значение искомого индекса.

Вот реализация алгоритма сортировки простыми вставками с использованием бинарного поиска:

```py
l = [8, 4, 7, 3, 1]

d = len(l)

for i in range(1, d): 
    elem = l[i]
    j = i

    if j >= 1 and l[j - 1] > elem:
        left, right = 0, i

        while left < right:
            mid = (left + right) // 2
            if l[mid] > elem:
                right = mid
            else:
                left = mid + 1

        while i > left:
            l[i] = l[i - 1]
            i -= 1

        l[left] = elem
        
print(l)
```

Как вы могли заметить начало алгоритма полностью совпадает с оригиналом, но способ поиска нужного индекса для вставки `elem` в список, реализован с помощью бинарного поиска, а смещение элементов цикла в правую сторону вынесено в отдельный цикл. Таким образом мы получаем тот же алгоритм, но работающий гораздо быстрее оригинального.

# Сборка мусора

**Сборщик мусора (Garbage Collector, GC)** — это встроенный механизм Python для автоматического управления памятью. Его задача — находить объекты, которые больше не используются программой, и удалять их, освобождая ресурсы.

**В Python управление памятью строится на двух уровнях:**

1. **Подсчет ссылок (Reference Counting)** — основной механизм.
У каждого объекта есть счетчик: сколько переменных или структур данных на него ссылаются. Как только счетчик опускается до нуля, объект удаляется из памяти мгновенно.

    ```py
    import sys
    x = [1, 2, 3]
    print(sys.getrefcount(x)) # Показывает количество ссылок — 2(перем. x и арг. x)
    y = x                     # Счетчик увеличился
    del y                     # Счетчик уменьшился
    ```

1. **Циклические ссылки и модуль `gc`** — вспомогательный механизм.
Подсчет ссылок не справляется, если два объекта ссылаются друг на друга (например, `A.ref = B` и `B.ref = A`), но при этом оба недоступны из основной программы. Для их поиска Python периодически запускает циклический сборщик мусора.

    - **Поколения (Generations)** : для оптимизации Python делит объекты на три поколения (0, 1 и 2). Новые объекты попадают в нулевое. Если объект сохранился после очистки, он переходит в более старшее поколение. Чем старше поколение, тем реже оно проверяется.

    - **Порог (Threshold)** : сборка запускается автоматически, когда количество созданных объектов в поколении превышает определенный лимит.

В отличие от языков вроде C, программисту в Python редко нужно вмешиваться в этот процесс. Однако через модуль `import gc` можно форсировать очистку командой `gc.collect()` или изменить параметры порогов.
