# Основные понятия

## Что такое модуль?

**Модуль** — в Python модулем называется библиотека функций, которые уже реализованы и готовы к использованию.

**Инструкция `import`** - инструкция, позволяющая подключить модуль и использовать его функции.

```py
import module # module - название модуля
```

Для того, чтобы **вызвать функцию** из модуля необходимо **обратиться к модулю** и указать название функции **через точку**:

```py
import module # импортируем модуль
module.func() # вызов функции
```

Стоит отметить, что если указанный атрибут модуля не будет найден, возникнет исключение `AttributeError`. А если не удастся найти модуль для импортирования, то `ModuleNotFoundError`.

Также для вывода всех функций, классов и переменных определенных в модуле можно использовать функцию [`dir()`](builtins.md#функции-отладки).

---

**В Python можно ипортировать функции из модулей и другими способами.**

**`from ... import *`** - импортирует все функции из заданного модуля в пространство именн программы. Это позволяет вызывать все функции модуля без обращнения к нему.

```py
from module import *
func() # можем вызвать функцию напрямую без указания модуля
```

Рекомендуется **избегать** импортирования через `from ... import *`, так как нет ясности, какие функции были импортированы и это **загрязняет пространство имён** лишними, неиспользуемыми функциями.

---

**`from ... import func_1, func_2`** - позволяет импортировать **конкретные** функции из заданного модуля в просранство именн программы. Это так же позволяет вызывать эти функции без обращнения к их модулю, однаком мы не можем использовать другие функции модуля, если они не указаны в импорте.
  
  ```py
  from module import func_1, func_2
  func_1()
  func_2()
  ```

---

Также ипмортируемым модулям можно давать **другие имена**, и обращаться к ним через эти имена.

**`import ... as name`** — позволяет импортировать модуль и обращаться к нему по имени `name`. То же самое работает и при импортировании отдельных функций модуля.

```py

import module as name
from module_1 import func as f

name.func()
f()

```

## Создание собственных модулей

Чтобы написать **собственный модуль** необходимо создать `.py` файл и импортировать его в свой основной код.

Пути поиска модулей указаны в переменной [`sys.path`](#модуль-sys). В него включены текущая директория и директории, в которых установлен Python.

Соответственно модуль можно оставить в папке с основной программой или разместить в другое место, но при условии что путь к модулю будет добавлен в `sys.path`.

**Правила именования модулей:**

- Модуль нельзя именовать так же, как и ключевое слово;

- Имена модулей нельзя начинать с цифры;

- Не нужно называть модуль так же, как какую-либо из встроенных функций. Это может создаст большие неудобства при его последующем использовании.

## Имя модуля — `__name__`

Чтобы узнать запущен модуль как самостоятельная программа или импортирован используется атрибут `__name__`.

```py
def main():
    print("Hello, World!")

if __name__ == "__main__":
    main()
```

В примере выше, если значением атрибута `__name__` будет строка `"__main__"`, значит скрипт запущен как самостоятельная программа и он выполнит необходимые операции. В данном случае вывоз функции `main()`, которая выведет приветствие в консоль.

Если же значением атрибута будет другое значение(название самого модуля), то скрипт будет исполнен как модуль, и функцию `main()` можно будет вызвать из другого скрипта.

Если не прописать такую конструкцию в файл модуля, то при его импорте другой программой он исполнится, как самостоятельная программа. Поэтому необходимо использовать `if __name__ == "__main__"`: в тех файлах, где есть какой-то «активный» код (принты, запуски функций, циклы), который не должен мешать при импорте.

## Пакеты

**Пакеты** – это способ иерархической организации модулей. Если говорить простыми словами, то это каталог в котором хранятся модули.

Главная цель пакетов - разместить модули в удобном порядке, чтобы сотни файлов не находились в одной папке и к ним легко можно было получить доступ для иморта.

Python версии ниже, чем 3.3 не признавал папку пакетом, если в ней не было файла `__init__.py`. Сейчас создавать файл `__init__.py` необязательно, но его рекомендуется создавать по двум причинам:

- Явное обозначение: чтобы и Python, и другие программисты сразу видели, что это папка с кодом, а не просто папка с картинками.
- Настройка импорта: если вы напишете код внутри `__init__.py`, он выполнится один раз в момент первого импорта любой части пакета. Например, с его помощью можно сократить пути импорта.

## Файлы байткода `.pyc`

Во время импорта модуля Python создает файлы байткода с расширением `.pyc`. Это необходимо для повышения производительности программы, и при импорте модуля в следующий раз, это сэкономит ресурсы и время.

Файлы байткода обычно хранятся в папке `__pycache__`, которая создается в том же каталоге, где расположены импортируемые модули. Если Python не сможет получить доступ для записи файлов в этот каталог, файлы `.pyc` созданы не будут.

## Модуль random

**Модуль random** — модуль, предоставляющий функции для генерации случайных чисел, букв и случайного выбора элементов последовательности (списка, строки и т.д.).

**Список основных функций модуля random:**

| Функция                           | Описание                                                                                     |
| --------------------------------- | ---------------------------------------------------------------------------------------------|
| `randint(a, b)`                   | Возвращает случайное целое число из отрезка [a; b] (включая оба конца).                      |
| `randrange(start, stop[, step])`  | Возвращает случайный элемент из последовательности, генерируемой `range(start, stop, step)`. |
| `random()`                        | Возвращает случайное число с плавающей точкой в диапазоне [0.0, 1.0) (1.0 не включается).    |
| `uniform(a, b)`                   | Возвращает случайное число с плавающей точкой в диапазоне [a, b] (или [b, a], если b < a).   |
| `seed(a=None)`                    | Устанавливает начальное значение (семя) для генератора псевдослучайных чисел.                |
| `shuffle(x)`                      | Перемешивает список на месте (изменяет исходный список).                                     |
| `choice(seq)`                     | Возвращает случайный элемент из непустой последовательности `seq` (списка, строки и т.д.)    |
| `sample(population, k)`           | Возвращает список из k уникальных случайных элементов из последовательности `population`.    |

Подробнее о нем можено прочитать в [официальной документации языка](https://docs.python.org/3.14/library/random.html).

## Модуль operator

**Модуль operator** — модуль, предоставляющий функции, соответствующие встроенным операторам языка и определяющий такие стандартные операции как сумма, произведение, проверка на равенство и т.д.

**Основные функций модуля operator:**

| Операция                   | Синтаксис      | Функция              |
|----------------------------|----------------|----------------------|
| Addition                   | `a + b`        | `add(a, b)`          |
| Containment Test           | `obj in seq`   | `contains(seq, obj)` |
| Division                   | `a / b`        | `truediv(a, b)`      |
| Division                   | `a // b`       | `floordiv(a, b)`     |
| Exponentiation             | `a ** b`       | `pow(a, b)`          |
| Modulo                     | `a % b`        | `mod(a, b)`          |
| Multiplication             | `a * b`        | `mul(a, b)`          |
| Negation (Arithmetic)      | `-a`           | `neg(a)`             |
| Subtraction                | `a - b`        | `sub(a, b)`          |
| Ordering                   | `a < b`        | `lt(a, b)`           |
| Ordering                   | `a <= b`       | `le(a, b)`           |
| Equality                   | `a == b`       | `eq(a, b)`           |
| Difference                 | `a != b`       | `ne(a, b)`           |
| Ordering                   | `a >= b`       | `ge(a, b)`           |
| Ordering                   | `a > b`        | `gt(a, b)`           |

Модуль operator реализован на языке C, поэтому функции этого модуля работают в разы быстрее, чем самописные функции в Python.

Подробнее о нем можено прочитать в [официальной документации языка](https://docs.python.org/3.14/library/operator.html).

## Модуль datetime

**Модуль datetime** — основной модуль для работы с датой и временем. Он позволяет создавать, форматировать и выполнять арифметические операции с временными объектами (например, прибавлять дни или вычислять разницу между датами).

Широко используется в разработке: от логирования событий до работы с базами данных и планирования задач.

**Основные классы:**

- `datetime.datetime` — комбинация даты и времени (год, месяц, день, час, минута, секунда);
- `datetime.date` — работа только с календарной датой;
- `datetime.time` — работа только со временем (независимо от даты);
- `datetime.timedelta` — класс для представления разницы между двумя моментами времени (используется для расчетов);

Подробнее о нем можно прочитать в [официальной документации языка](https://docs.python.org/3.14/library/datetime.html) и в этой [главе](advanced.md#дата-и-время-в-python).

## Модуль cmath

**Модуль cmath (complex math)** — модуль, предоставляющий обширный функционал для выполнения математических операций с [комплексными числами](types.md#комплексные-числа).

**Основные особенности cmath:**

- Работает только с комплексными числами.На вход можно передать и вещественные (`float`, `int`), они автоматически преобразуются к комплексным (`x + 0j`).
- Возвращает всегда комплексный результат, даже если мнимая часть равна нулю.

**Основные функции модуля cmath:**

| Функция                                       | Описание                                           |
|-----------------------------------------------|----------------------------------------------------|
| `cmath.sqrt(z)`                               | Квадратный корень                                  |
| `cmath.exp(z)`                                | Экспонента `e^z`                                   |
| `cmath.log(z[, base])`                        | Натуральный логарифм (или по указанному основанию) |
| `cmath.sin(z), cmath.cos(z), cmath.tan(z)`    | Тригонометрические функции                         |
| `cmath.asin(z), cmath.acos(z), cmath.atan(z)` | Обратные тригонометрические функции                |
| `cmath.sinh(z), cmath.cosh(z), cmath.tanh(z)` | Гиперболические функции                            |

**Полезные константы:**

- `cmath.pi` — `π`
- `cmath.e` — основание натурального логарифма
- `cmath.tau` — `τ` (`2π`)
- `cmath.inf` — комплексная бесконечность (`inf + 0j`)
- `cmath.nan` — комплексное «не число» (`nan + 0j`)

Константы возвращают вещественные числа типа `float`.

Подробнее о нем можено прочитать в [официальной документации языка](https://docs.python.org/3.14/library/cmath.html).

## Модуль tkinter

**Модуль tkinter** — стандартная библиотека Python для создания графических интерфейсов (GUI). Это обертка над графическим движком Tcl/Tk, которая поставляется вместе с интерпретатором «из коробки».

Предназначен для создания простых десктопных приложений с окнами, кнопками, текстовыми полями и меню. Это самый быстрый способ сделать визуальную программку без установки тяжелых сторонних библиотек (вроде PyQt).

**Основные возможности:**

- `Tk()` — создание главного (корневого) окна приложения;
- `Label / Button / Entry` — базовые виджеты (текст, кнопка, поле ввода);
- `pack() / grid() / place()` — менеджеры геометрии, отвечающие за то, как элементы располагаются в окне;
- `mainloop()` — запуск бесконечного цикла обработки событий (нажатий клавиш, кликов мыши);

Подробнее о нем можно прочитать в [официальной документации языка](https://docs.python.org/3.14/library/tkinter.html).

## Модуль sys

**Модуль sys** — модуль, предоставляющий доступ к некоторым переменным и функциям, взаимодействующим с интерпретатором Python. Он позволяет управлять поведением среды выполнения и получать информацию о системе.

Активно используется в реальной разработке для написания скриптов, обработки аргументов командной строки и управления завершением работы программ.

**Основные возможности:**

- `sys.argv` — список аргументов командной строки, переданных скрипту;
- `sys.exit()` — корректный выход из программы с возможностью передачи кода возврата;
- `sys.path` — список путей, где Python ищет модули для импорта (можно менять вручную);
- `sys.stdin/stdout/stderr` — управление стандартными потоками ввода, вывода и ошибок;

Подробнее о нем можно прочитать в [официальной документации языка](https://docs.python.org/3.14/library/sys.html).

## Модуль json

**Модуль json** — стандартный инструмент для кодирования и декодирования данных в формате JSON (JavaScript Object Notation). Он преобразует объекты Python (словари, списки) в текстовую строку и обратно.

Является фундаментальным в веб-разработке: используется для обмена данными между сервером и браузером, работы с API и хранения конфигурационных файлов.

**Основные функции:**

- `json.dumps()` — преобразует объект Python в строку формата JSON;
- `json.loads()` — преобразует строку JSON обратно в объект Python;
- `json.dump()` — записывает данные JSON напрямую в файл;
- `json.load()` — считывает данные JSON напрямую из файла;

Подробнее о нем можно прочитать в [официальной документации языка](https://docs.python.org/3.14/library/json.html) и в этой [главе](advanced.md#формат-json).

## Модуль pickle

**Модуль pickle** — стандартный инструмент для сериализации и десериализации объектов Python. Он превращает любой объект (список, словарь, даже целый класс) в бинарный поток байтов и обратно.

В отличие от JSON, который хранит только простые данные (строки, числа), pickle может сохранить почти любой сложный объект со всеми его связями и внутренним состоянием.

>Важное ограничение: Данные pickle небезопасны. Никогда не распаковывайте файлы, полученные из ненадежных источников, так как это может привести к выполнению произвольного кода.

**Основные функции:**

- `pickle.dumps()` — превращает объект в строку байтов;
- `pickle.loads()` — восстанавливает объект из строки байтов;
- `pickle.dump()` — записывает объект напрямую в бинарный файл;
- `pickle.load()` — считывает объект напрямую из бинарного файла;

Подробнее о нем можно прочитать в [официальной документации языка](https://docs.python.org/3.14/library/pickle.html).

## Модуль io

**Модуль io** — основной модуль для работы с потоками ввода-вывода (input/output). Он предоставляет инструменты для манипуляции данными в памяти так, будто это обычные файлы (текстовые или бинарные).

Широко используется в разработке для тестирования, формирования временных буферов и оптимизации работы с данными без обращения к реальному диску.

**Основные классы:**

- `io.StringIO` — виртуальный файл в оперативной памяти для текстовых данных;
- `io.BytesIO` — виртуальный файл в оперативной памяти для бинарных данных (изображения, архивы);
- `io.TextIOWrapper` — обертка для управления кодировкой и переносами строк в буферах;

Подробнее о нем можно прочитать в [официальной документации языка](https://docs.python.org/3.14/library/io.html).

## Модуль os

**Модуль os** — базовый модуль для взаимодействия с операционной системой. Он предоставляет функции для работы с файловой системой, переменными окружения и процессами на низком уровне.

Это незаменимый инструмент системного программиста, хотя для работы именно с путями в современном коде его часто вытесняет pathlib.

**Основные возможности:**

- `os.environ` — доступ к переменным окружения (например, os.environ.get('HOME'));
- `os.listdir()` — получение списка всех файлов и папок в директории;
- `os.remove() / os.rmdir()` — удаление файлов или пустых папок;
- `os.system()` — выполнение команд в терминале (но лучше использовать subprocess);
- `os.name` — определение типа ОС (posix для Linux/macOS или nt для Windows);

Подробнее о нем можно прочитать в [официальной документации языка](https://docs.python.org/3.14/library/os.html).

## Модуль pathlib

**Модуль pathlib** — современный стандарт для работы с путями файловой системы через объектно-ориентированный подход. Он заменяет старые функции из os.path, представляя пути не как обычные строки, а как объекты.

Считается лучшим выбором в современной разработке благодаря читаемости кода и кроссплатформенности (автоматически подбирает правильные слэши для Linux/Windows).

**Основные возможности:**

- `Path.cwd()` — возвращает текущую рабочую директорию;
- `Path.home()` — возвращает путь к домашней папке пользователя;
- `path.exists()` — проверяет, существует ли файл или папка;
- `path.read_text() / path.write_text()` — чтение и запись файлов одной командой (без with open(...));
- `path.mkdir(parents=True)` — создание вложенных папок одной командой;

Подробнее о нем можно прочитать в [официальной документации языка](https://docs.python.org/3.14/library/pathlib.html).

## Модуль logging

**Модуль logging** — стандартный гибкий инструмент для отслеживания событий во время работы программы. Он заменяет обычный print(), позволяя разделять сообщения по уровням важности и записывать их в разные места (консоль, файлы, внешние серверы).

Является обязательным стандартом в профессиональной разработке: без него невозможно эффективно искать ошибки в работающем сервисе или серверном приложении.

**Уровни логирования (от меньшего к большему):**

- `DEBUG` — детальная информация для диагностики;
- `INFO` — подтверждение того, что всё идет по плану;
- `WARNING` — индикация чего-то неожиданного (но программа еще работает);
- `ERROR` — серьезная проблема, часть функций не выполнена;
- `CRITICAL` — фатальная ошибка, программа может завершиться;

**Основные возможности:**

- `basicConfig()` — быстрая настройка формата вывода и файла;
- `getLogger()` — создание именованных логгеров для разных модулей проекта;
- `FileHandler / StreamHandler` — отправка логов одновременно в файл и в консоль;

Подробнее о нем можно прочитать в [официальной документации языка](https://docs.python.org/3.14/library/logging.html).

## Модуль platform

**Модуль platform** — стандартный модуль для извлечения информации о базовой платформе (операционной системе, оборудовании и версии интерпретатора). Он позволяет программе понимать, в какой среде она запущена, и адаптировать своё поведение.

Незаменим при создании кроссплатформенного софта, когда нужно прописать разные пути или команды для Linux, Windows и macOS.

**Основные возможности:**

- `platform.system()` — возвращает название ОС (Linux, Windows, Darwin);
- `platform.release()` — возвращает версию релиза (например, версию ядра 6.8.0-40-generic);
- `platform.machine()` — тип архитектуры процессора (например, x86_64 или arm64);
- `platform.python_version()` — возвращает строку с текущей версией Python (например, 3.12.3);
- `platform.uname()` — возвращает сводный объект со всеми данными (система, узел, релиз, версия, машина);

Подробнее о нем можно прочитать в [официальной документации языка](https://docs.python.org/3.14/library/platform.html).

## Модуль time

**Модуль time** — стандартный модуль для работы с системным временем и низкоуровневыми счетчиками. В отличие от datetime, который работает с «человеческими» датами, time оперирует секундами, прошедшими с начала Эпохи (1 января 1970 года).

Является незаменимым инструментом для профилирования кода (замера скорости), создания пауз в выполнении программ и синхронизации процессов.

**Основные возможности:**

- `time.sleep(secs)` — приостанавливает выполнение программы на заданное количество секунд (можно дробные, например 0.1);
- `time.time()` — возвращает текущее время в секундах (Unix Timestamp). Удобно для вычисления разницы "было/стало";
- `time.perf_counter()` — наиболее точный счетчик для замера времени выполнения кода (не зависит от системных часов);
- `time.localtime()` — преобразует секунды в структуру с понятными полями (год, месяц, час и т.д.);

Подробнее о нем можно прочитать в [официальной документации языка](https://docs.python.org/3.14/library/time.html).
