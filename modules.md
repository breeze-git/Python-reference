# Основные понятия

## Что такое модуль?

**Модуль** — в Python модулем называется библиотека функций, которые уже реализованы и готовы к использованию.

**Инструкция `import`** - инструкция, позволяющая подключить модуль и использовать его функции.

```py
import module # module - название модуля
```

Для того, чтобы **вызвать функцию** из модуля необходимо **обратиться к модулю** и указать название функции **через точку**:

```py
import module # импортируем модуль
module.func() # вызов функции
```

Стоит отметить, что если указанный атрибут модуля не будет найден, возникнет исключение `AttributeError`. А если не удастся найти модуль для импортирования, то `ModuleNotFoundError`.

Также для вывода всех функций, классов и переменных определенных в модуле можно использовать функцию [`dir()`](builtins.md#функции-отладки).

---

**В Python можно ипортировать функции из модулей и другими способами.**

**`from ... import *`** - импортирует все функции из заданного модуля в пространство именн программы. Это позволяет вызывать все функции модуля без обращнения к нему.

```py
from module import *
func() # можем вызвать функцию напрямую без указания модуля
```

Рекомендуется **избегать** импортирования через `from ... import *`, так как нет ясности, какие функции были импортированы и это **загрязняет пространство имён** лишними, неиспользуемыми функциями.

---

**`from ... import func_1, func_2`** - позволяет импортировать **конкретные** функции из заданного модуля в просранство именн программы. Это так же позволяет вызывать эти функции без обращнения к их модулю, однаком мы не можем использовать другие функции модуля, если они не указаны в импорте.
  
  ```py
  from module import func_1, func_2
  func_1()
  func_2()
  ```

---

Также ипмортируемым модулям можно давать **другие имена**, и обращаться к ним через эти имена.

**`import ... as name`** — позволяет импортировать модуль и обращаться к нему по имени `name`. То же самое работает и при импортировании отдельных функций модуля.

```py

import module as name
from module_1 import func as f

name.func()
f()

```

## Создание собственных модулей

Чтобы написать **собственный модуль** необходимо создать `.py` файл и импортировать его в свой основной код.

Пути поиска модулей указаны в переменной [`sys.path`](#модуль-sys). В него включены текущая директория и директории, в которых установлен Python.

Соответственно модуль можно оставить в папке с основной программой или разместить в другое место, но при условии что путь к модулю будет добавлен в `sys.path`.

**Правила именования модулей:**

- Модуль нельзя именовать так же, как и ключевое слово;

- Имена модулей нельзя начинать с цифры;

- Не нужно называть модуль так же, как какую-либо из встроенных функций. Это может создаст большие неудобства при его последующем использовании.

## Имя модуля — `__name__`

Чтобы узнать запущен модуль как самостоятельная программа или импортирован используется атрибут `__name__`.

```py
def main():
    print("Hello, World!")

if __name__ == "__main__":
    main()
```

В примере выше, если значением атрибута `__name__` будет строка `"__main__"`, значит скрипт запущен как самостоятельная программа и он выполнит необходимые операции. В данном случае вывоз функции `main()`, которая выведет приветствие в консоль.

Если же значением атрибута будет другое значение(название самого модуля), то скрипт будет исполнен как модуль, и функцию `main()` можно будет вызвать из другого скрипта.

Если не прописать такую конструкцию в файл модуля, то при его импорте другой программой он исполнится, как самостоятельная программа. Поэтому необходимо использовать `if __name__ == "__main__"`: в тех файлах, где есть какой-то «активный» код (принты, запуски функций, циклы), который не должен мешать при импорте.

## Пакеты

**Пакеты** – это способ иерархической организации модулей. Если говорить простыми словами, то это каталог в котором хранятся модули.

Главная цель пакетов - разместить модули в удобном порядке, чтобы сотни файлов не находились в одной папке и к ним легко можно было получить доступ для иморта.

Python версии ниже, чем 3.3 не признавал папку пакетом, если в ней не было файла `__init__.py`. Сейчас создавать файл `__init__.py` необязательно, но его рекомендуется создавать по двум причинам:

- Явное обозначение: чтобы и Python, и другие программисты сразу видели, что это папка с кодом, а не просто папка с картинками.
- Настройка импорта: если вы напишете код внутри `__init__.py`, он выполнится один раз в момент первого импорта любой части пакета. Например, с его помощью можно сократить пути импорта.

## Файлы байткода `.pyc`

Во время импорта модуля Python создает файлы байткода с расширением `.pyc`. Это необходимо для повышения производительности программы, и при импорте модуля в следующий раз, это сэкономит ресурсы и время.

Файлы байткода обычно хранятся в папке `__pycache__`, которая создается в том же каталоге, где расположены импортируемые модули. Если Python не сможет получить доступ для записи файлов в этот каталог, файлы `.pyc` созданы не будут.

## Модуль decimal

**Тип данных `Decimal`** – тип для **точной десятичной арифметики без ошибок округления**. Он представляет собой число с плавающей точкой и используется для точных вычислений и финансовых расчётов.

Создать `Decimal` число можно из обычного целого числа (`int`) или из строки (`str`). Возможно создание из числа с плавающей точкой (`float`), однако делать этого не рекомендуется, так как в `Decimal` попадет уже **неправильно округленное** число.

```py
import decimal

num1 = decimal.Decimal('0.3')

num2 = decimal.Decimal(0.3)

print(num1, num2, sep = '\n')

0.3 # из str
0.299999999999999988897769753748434595763683319091796875 # из float
```

**Свойства `Decimal` чисел:**

- С `Decimal` числами работают все привычные операции: сложение, вычитание, умножение, деление, возведение в степень.

- `Decimal` **неизменяемый**. Операции над ним приводят к созданию новых объектов, при этом старые не меняются.

- `Decimal` числа можно сравнивать между собой, как обычные числа, причем в отличие от `float` чисел допускается и **точное равенство**.

---
Базовые параметры `Decimal` можно посмотреть в его **контексте**, выполнив функцию `getcontext()`.

```py
import decimal

print(decimal.getcontext())

Context(prec=28, rounding=ROUND_HALF_EVEN, Emin=-999999, Emax=999999, capitals=1, clamp=0, flags=[FloatOperation], traps=[InvalidOperation, DivisionByZero, Overflow])
```

**Контекст** – объект, содержащий настройки для выполнения операций.

**Параметры:**

- `prec` — точность (количество значащих цифр)

- `rounding` — режимы округления
  
  Основные из них:
  
  - `ROUND_HALF_UP` — к ближайшему, `0.5` → вверх;
  - `ROUND_HALF_EVEN` — «банковское» округление;
  - `ROUND_DOWN` — усечение;
  - `ROUND_UP` — всегда вверх;
  - `ROUND_CEILING`, `ROUND_FLOOR` — к `+∞` / `−∞`.

- `Emin`, `Emax`  — минимальный и максимальный показатель степени;

- `capitals` — регистр для `Infinity` и `NaN`:
  
  - `1` — `Infinity`, `NaN`
  - `0` — `infinity`, `nan`

- `flags`, `traps` — флаги и ловушки:

  - `DivisionByZero` — Деление на ноль
  - `Overflow` — Переполнение
  - `Inexact` — Потеря точности
  - `InvalidOperation` — Недопустимая операция
  - и др.

**Контекст** позволяет точно контролировать поведение вычислений:

- Для финансовых расчётов: фиксированная точность, округление `ROUND_HALF_UP`.
- Для научных вычислений: высокая точность, обработка переполнений.
- Для отладки: включить все ловушки, чтобы ловить неточности.

---

**Основные математические методы `Decimal`:**

| Метод     | Описание                                                          |
|-----------|-------------------------------------------------------------------|
| `sqrt()`  | вычисляет квадратный корень из `Decimal` числа                    |
| `exp()`   | возвращает `e^x` для `Decimal` числа                              |
| `ln()`    | вычисляет натуральный логарифм (по основанию `e`) `Decimal` числа |
| `log10()` | вычисляет десятичный логарифм (по основанию `10`) `Decimal` числа |

---

**Метод `d.as_tuple()`** — возвращает кортеж из `3` элементов:

- `sign` – знак числа (`0` для положительного числа и `1` для отрицательного числа);

- `digits` – цифры числа;

- `exponent` – значение экспоненты.

```py
from decimal import Decimal

d = Decimal('3.2547')

print(d.as_tuple())

DecimalTuple(sign=0, digits=(3, 2, 5, 4, 7), exponent=-4)
```

**Метод `d.quantize(exp)`** — округляет десятичное число `d` так, чтобы оно имело такую же экспоненту (разрядность), как у аргумента `exp`.

**Округление до 2 знаков после запятой:**

```py
from decimal import Decimal

d = Decimal('3.14159')
rounded = price.quantize(Decimal('0.01'))  # '0.01' = два знака после запятой
print(rounded)  # 3.14
```

## Модуль fractions

**Тип данных `Fraction`** — тип данных модуля `fractions`, представляющий рациональное число в виде **несократимой обыкновенной дроби с целыми числителем и знаменателем**. Обеспечивает **точную арифметику без ошибок округления**, присущих числам с плавающей точкой.

Создать Fraction число можно несколькими способами:

- из целых чисел, передав значения числителя и знаменателя дроби:

  ```py
  from fractions import Fraction

  num = Fraction(1, 3)
  print(num)

  1/3
  ```

- из строки на основании десятичного представления:

  ```py
  from fractions import Fraction

  num = Fraction('0.5')
  print(num)

  1/2
  ```
  
- из строки на основании обыкновенной дроби:

  ```py
  from fractions import Fraction

  num = Fraction('3/9')
  print(num)

  1/3
  ```

  Обратите внимание на то, что при создании рационального числа `Fraction`, автоматически происходит **сокращение числителя и знаменателя** дроби.

Возможно создание из числа с плавающей точкой (`float`), однако делать этого не рекомендуется, так как в `Fraction` попадет уже **неправильно округленное** число, что может привести к непредсказуемому результату.

**Свойства `Fraction` чисел:**

- С `Fraction` числами работают все привычные операции: сложение, вычитание, умножение, деление, возведение в степень.

- `Fraction` числа можно сравнивать между собой точно так же, как и любые другие числа.

- `Fraction` **неизменяемый**. Операции над ним приводят к созданию новых объектов, при этом старые не меняются.

- `Fraction` числа можно передавать как аргументы функций, ожидающих `float`. Тогда они будут преобразованы во `float`. К примеру, модуль `math`, оперирующий `float` числами, может работать и с `Fraction` числами. Результатом работы этих функций будут `float` числа.

- В Python нельзя совершать арифметические операции между типами `Decimal` и `Fraction`.

---

**Свойства numerator и denominator**:

Для получения **числителя** и **знаменателя** `Fraction` числа **используются свойства** `numerator` и `denominator`.

```py
from fractions import Fraction

num = Fraction('3/7')

print(num.numerator, num.denominator)

3 7
```

---

**Метод `f.as_integer_ratio()`** — возвращает кортеж, состоящий из числителя и знаменателя данного `Fraction` числа.

```py
from fractions import Fraction

num = Fraction('3/7')

print(num.as_integer_ratio())

(3, 7)
```

**Метод `f.limit_denominator()`** — возвращает самую близкую к данному числу рациональную дробь, чей знаменатель не превосходит переданного аргумента.

```py
from fractions import Fraction

f = Fraction('0.3333333333333333')

simple = f.limit_denominator(100)
print(f, simple, sep = '\n')

3333333333333333/10000000000000000
1/3
```

## Модуль cmath

**Модуль cmath (complex math)** — модуль, предоставляющий обширный функционал для выполнения математических операций с [комплексными числами](types.md#комплексные-числа).

**Основные особенности cmath:**

- Работает только с комплексными числами.На вход можно передать и вещественные (`float`, `int`), они автоматически преобразуются к комплексным (`x + 0j`).
- Возвращает всегда комплексный результат, даже если мнимая часть равна нулю.

**Основные функции модуля cmath:**

| Функция                                       | Описание                                           |
|-----------------------------------------------|----------------------------------------------------|
| `cmath.sqrt(z)`                               | Квадратный корень                                  |
| `cmath.exp(z)`                                | Экспонента `e^z`                                   |
| `cmath.log(z[, base])`                        | Натуральный логарифм (или по указанному основанию) |
| `cmath.sin(z), cmath.cos(z), cmath.tan(z)`    | Тригонометрические функции                         |
| `cmath.asin(z), cmath.acos(z), cmath.atan(z)` | Обратные тригонометрические функции                |
| `cmath.sinh(z), cmath.cosh(z), cmath.tanh(z)` | Гиперболические функции                            |

**Полезные константы:**

- `cmath.pi` — `π`
- `cmath.e` — основание натурального логарифма
- `cmath.tau` — `τ` (`2π`)
- `cmath.inf` — комплексная бесконечность (`inf + 0j`)
- `cmath.nan` — комплексное «не число» (`nan + 0j`)

Константы возвращают вещественные числа типа `float`.

## Модуль turtle

**Модуль turtle** — модуль, **реализующий концепцию «черепашьей графики»** (turtle graphics). Он предназначен для **образовательных целей**: наглядного изучения основ программирования (циклов, функций, условий, координат) через визуальный результат.

Не используется в реальной разработке. Это учебный инструмент.

**Основная идея:**

- Представьте черепашку с кисточкой на спине, которая ползает по экрану.
- Когда перо опущено — **она рисует линию**.
- Вы управляете черепашкой командами: **двигаться, поворачивать, поднимать/опускать перо**.

**Вот некоторые из основных функций модуля turtle:**

**`turtle.showturtle()`** — показывает черепашку (курсор) в графическом окне.

---

**`turtle.forward(n)`** — перемещает черепашку вперед на `n` пикселей.

---

**`turtle.backward(n)`** — перемещает черепашку назад на `n` пикселей.

---

**`turtle.right(angle)`** — поворачивает черепашку вправо на `angle` градусов.

---

**`turtle.left(angle)`** — поворачивает черепашку влево на `angle` градусов.

---

**`turtle.setheading(angle)`** — устанавливает угловое направление черепашки на `angle` градусов.

---

**`turtle.heading()`** —  применяется для определения текущего углового направления черепашки.

---

**`turtle.shape()`** — применяется для изменения внешнего вида черепашки. Команда принимает в качестве аргумента строковое название фигуры, определяющей форму черепашки.

Доступные фигуры:

- square (квадрат);
- arrow (стрелка);
- circle (круг);
- turtle (черепашка);
- triangle (треугольник);
- classic (классическая стрелка).

---

**`turtle.penup()`** — поднимает перо.

---

**`turtle.pendown()`** – опускает перо.

---

**`turtle.circle(r)`** — применяется для рисования круга радиусом `r`.

---

**`turtle.dot(size, color)`** — применяется для рисования точки размером `size` и цвета `color` (по умолчанию равен `black`).

---

**`turtle.pensize(width)`** — устанавливает ширину пера в `width` пикселей.

---

**`turtle.pencolor()`** — применяется для изменения цвета линий. В качестве аргумента можно передать название цвета строкой или цвет в формате RGB — [кортеж](tuples.md#что-такое-кортеж) из трёх чисел или три отдельных числа.

---

**`turtle.Screen().bgcolor()`** — применяется для изменения цвета фона. В качестве аргумента можно передать название цвета строкой или цвет в формате RGB — кортеж из трёх чисел или три отдельных числа.

---

**`turtle.Screen().bgpic(filename)`** — применяется для установки фонового изображения в графическом окне.

---

**`turtle.stamp()`** —  оставляет штамп черепашки.

---

**`turtle.clear()`** — стирает все рисунки в графическом окне. Но не меняет положение черепашки, цвет рисунка и цвет фона графического окна.

---

**`turtle.reset()`** — стирает все рисунки, имеющиеся в графическом окне, задает черный цвет рисунка и возвращает черепашку в исходное положение в центре экрана. Эта команда не переустанавливает цвет фона графического окна.

---

**`turtle.clearscreen()`** — стирает все рисунки в графическом окне, меняет цвет рисунка на черный, а цвет фона на белый, и возвращает черепашку в исходное положение в центре графического окна.

---

**`turtle.Screen().setup(a, b)`** — устанавливает размер графического окна a × b пикселей.

---

**`write()`** — применяется для вывода текста в графическое окно. Левый нижний угол первого символа выведенного текста будет расположен в точке с координатами черепашки.

Аргументы команды:

- `arg` – текст, который нужно вывести;
- `move` – указывает будет ли двигаться черепашка по мере рисования надписи (по умолчанию значение `False`);
- `align` – служит для выравнивания надписи относительно черепашки, может принимать три строковых значения `right`, `center`, `left` (по умолчанию значению `right`);
- `font` – [кортеж](tuples.md#что-такое-кортеж) из трех значений: (название шрифта, размер шрифта, тип начертания). В качестве начертания можно использовать строковые значения: `normal` — обычный, `bold` — полужирный, `italic` — курсив, или объединить два последних, тогда текст будет напечатан полужирным курсивом.

---

**`turtle.begin_fill()`, `turtle.end_fill()`** — применяются для заливки геометрической фигуры цветом. `turtle.begin_fill()` применяется до начертания фигуры, а после завершения начертания используется команда `turtle.end_fill()` и геометрическая фигура заполняется текущим цветом заливки.

---

**`fillcolor()`** —  меняет цвет заливки. Аргумент команды — название цвета в виде строкового литерала, либо значения трех компонентов RGB.

---

**`turtle.Turtle()`** — создаёт новый экземпляр класса `Turtle()`. Позволяет использовать несколько черепашек в одной программе.

---

**`turtle.tracer(n, delay)`** — включает/выключает анимацию черепашки и устанавливает задержку для обновления рисунков.

---

**`onkey(fun, key)`** — используется для отслеживания нажатия клавиш клавиатуры. Она связывает функцию обратного вызова `fun` с событием нажатия клавиши `key`.

---

**`turtle.Screen().listen()`** — устанавливает фокус на экран черепашки.

---

**`onclick(fun)`** — используется для отслеживания нажатия мыши. Она связывает функцию обратного вызова `fun` с событием нажатия левой кнопки мыши.
