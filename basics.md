# Введение в Python

## Язык программирования Python

**Python** — это высокоуровневый, интерпретируемый язык программирования с *[динамической типизацией](glossary.md#динамическая-типизация)*, предназначенный для создания приложений различных типов.

## Основные особенности языка программирования Python

- **Скриптовый язык**. Код программ определяется в виде скриптов.

- **Поддержка самых различных парадигм программирования**, в том числе объектно-ориентированной и функциональной парадигм.

- **Интерпретация программ**. Для работы со скриптами необходим интерпретатор, который запускает и выполняет скрипт.

- **Портативность и платформонезависимость**. Не имеет значения, какая операционная система — Windows, Mac OS, Linux, скрипт будет запускаться на всех этих ОС при наличии интерпретатора.

- **Автоматическое управление памяти**

- **Динамическая типизация**

- **Большое количество пакетов и библиотек**

# Основы синтаксиса и структуры программы

## Инструкции и регистрозависимость

**Инструкция** — это команда интерпретатору на выполнение какого либо действия. Каждая инструкция размещается на новой строке. Программы на языке Python состоят из набора инструкций.

Python — **регистрозависимый язык**. Если вместо метода `print` для вывода на консоль мы попробуем использовать метод `Print`, то интерпретатор выдаст ошибку.

## Блок кода

**Блок кода** — это группа инструкций, объедененных логически и синтаксически, и выполняющихся как единое целое в определённом контексте программы.

В Python блоки кода синтаксически обозначаются с помощью **отступов**. Согласно руководству по оформлению кода, отступ желательно делать из 4 пробелов.

## Комментарии

**Комментарии** в Python бывают **блочные** и **строчные**:

- строчные — `#`

- блочные — `''' '''`

## Стиль кода в Python и PEP 8

**PEP 8** — это официальный **руководство по стилю кода в Python**, которое описывает рекомендации по оформлению кода, чтобы он был **читаемым и единообразным**.

**Основные рекомендации:**

1. **Отступы**

   - Используется **4 пробела** на уровень вложенности.
   - Табуляция (`\t`) не рекомендуется.

2. **Длина строки**

   - Рекомендуется не превышать **79 символов**.
   - Для длинных выражений можно использовать перенос с `\` или скобки.

3. **Пробелы**

   - После запятой в списках: `a, b, c`
   - Около операторов присваивания и арифметических: `x = a + b`
   - Не ставить лишние пробелы внутри скобок: `func(a, b)` — правильно, `func( a, b )` — неправильно

4. **Именование**

   - Переменные и функции: `snake_case` (например, `my_variable`)
   - Классы: `CamelCase` (например, `MyClass`)
   - Константы: `UPPER_CASE` (например, `MAX_SIZE`)

5. **Комментарии**

   - Использовать **ясные и лаконичные комментарии**.
   - Строки документации (`docstring`) для функций, классов и модулей — тройные кавычки `""" """`.

6. **Импорты**

   - Каждый импорт в новой строке:

     ```py
     import os
     import sys
     ```

   - Сначала стандартные библиотеки, потом сторонние, потом свои модули.

# Переменные

## Что такое переменная?

**Переменная** — это именованный контейнер, предназначенный для хранения данных.

## Именование переменных

**Название переменной:**

- должно начинаться с буквы или со знака подчеркивания.
- может содержать буквы, цифры и знак подчеркивания.
- не должно совпадать с названиями ключевых слов языка Python.
- применяется два типа наименования переменных: camel case и underscore notation.
- чувствительно к регистру(`name ≠ Name`);

## Присваивание

Символ `=` в языках программирования выступает в роли **оператора присваивания**. Он выполняет задачу связывания значения, расположенного справа от него, с именем переменной, указанной слева.

```py
num = 13
```

**Множественное присваивание** — это одновременное присваивание значений нескольким переменным. Слева от знака `=` указываются имена переменных через запятую, а справа — значения, тоже через запятую. Переменным присваиваются значения в том порядке, в котором они прописаны.

```py
name, surname = 'Will', 'Turner'
print('Имя:', name, 'Фамилия:', surname)
Имя: Will Фамилия: Turner
```
