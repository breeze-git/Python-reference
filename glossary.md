# Глоссарий

**<b id="dynamic-typing">Динамическая типизация</b>** — переменная **не привязана к конкретному типу данных и может хранить значения любого типа.** Тип переменной определяется из присвоенных ей данных и **может изменяться при каждом новом присваивании.**

**<b id="strict-typing">Строгая типизация</b>** — требует жесткого соответствия типов данных при операциях и запрещает их автоматическое неявное смешивание. Например, если переменная объявлена как число, язык не позволит прибавить к ней строку, без явного преобразования типов, что предотвращает ошибки.

## Блок кода

**<b id="#code-block">Блок кода</b>** — это группа инструкций, объедененных **логически** и **синтаксически**, и выполняющихся как **единое целое** в определённом контексте программы.

## Управляющая конструкция

**<b id="#control-structure">Управляющая конструкция</b>** — это конструкция, которая **управляет порядком** выполнения кода (потоком выполнения программы).

## Итерация

**Итерация** — однократное выполнение тела цикла.

## Итератор

**<b id="#iterator">Итератор</b>** — это объект в Python, который реализует **протокол итерации**, то есть умеет **последовательно возвращать элементы из некоторой коллекции или последовательности**.

**Особенности:**

- итераторы можно обойти циклом `for`;
- итератор можно преобразовать в список или кортеж, с помощью функций `list()` и `tuple()`;
- итератор можно распаковать с помощью оператора распаковки `*`.

## Итерируемый объект

**Итерируемый объект** — это объект в языке Python, который реализует метод `__iter__()`, возвращающий итератор, либо метод `__getitem__()` с поддержкой целочисленной индексации, позволяющий последовательно получать свои элементы.

## Структура данных

**Структура данных (data structure)** — это **способ организации и хранения данных в памяти**, определяющий,
как данные расположены и какие операции над ними можно выполнять эффективно.

**<b id="#collection">Коллекция</b>** — это структура данных, предназначенная **для хранения, организации и обработки набора значений**.

## Индексация элементов

**<b id="#indexing">Индекс</b>** — **порядковый номер** элемента, находящегося в **составе структуры данных**.

В Python **индексация начинается с** `0`. То есть первый элемент имеет индекс `[0]`, второй `[1]` и т.д.

**<b id="#method">Метод</b>** — это **функция**, определённая **внутри класса**. При вызове через объект она **автоматически получает этот объект первым аргументом**.

**Синтаксис вызова:** `object.method()`

```py
s = 'Python'
print(s.upper())

PYTHON
```

В примере выше мы вызываем метод `upper()` принадлежащий классу `str` на переменной `s`. В результате метод возвращает новую строку `PYTHON`.

## Система счисления

**<b id="#number-system">Система счисления</b>** — символический метод записи чисел, то есть представление чисел с помощью письменных знаков.

В Python целые числа можно записывать в **десятичной** (по умолчанию), **двоичной** (`0b`), **восьмеричной** (`0o`) и **шестнадцатеричной** (`0x`) системах счисления. Основание системы указывает, сколько цифр используется для записи чисел (2, 8, 10, 16).

## Поверхностная копия

**<b id="#shallow-copy">Поверхностная копия</b>** — копия какого либо объекта без **полного** копирования вложенных коллекций, если таковые имеются. То есть, если **исходный объект содержит вложенные изменяемые коллекции** (например, списки или словари), то в копии **будут ссылки на те же самые вложенные объекты, а не их независимые копии.**

```py
lst = [1, 2, 3, 4, ['one', 'two']]
print(lst)

lst_copy = lst.copy()
lst[4][0] = 'three'

print(lst, lst_copy, sep = '\n')

[1, 2, 3, 4, ['three', 'two']] # оригинальный список изменился
[1, 2, 3, 4, ['three', 'two']]

```

**Debug** — применительно к компьютерным программам обозначает **отладку** (обнаружение и устранение ошибок, которые при этом принято называть "bug", т.е. "жук"). По всей видимости, это берёт своё начало с процедуры изгнания насекомых из схем больших ЭВМ.
